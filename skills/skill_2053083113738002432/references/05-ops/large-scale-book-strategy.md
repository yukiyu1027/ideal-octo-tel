# 大规模书稿应对策略（M1 / M2 / M3）

**版本**：2.0.2 · **权威来源**：[`section-3-workflow.md`](../01-core/section-3-workflow.md) 正文目录节

> **与 SKILL 档位单一真源**：[`scale-tiers.json`](./scale-tiers.json)（`manuscriptMagnitude`）。M1/M2/M3 为正文 workflow 惯用称呼；与 **S/M/L/XL 写作策略档位** 的映射见该文件，避免与「成稿字符分档（xl-project-init）」混淆。

## 规模分级（与第3节一致）

| 级别 | 字数规模约定 | 适用场景 |
|------|------------|----------|
| **M1** | 2万~5万字 / 1.5万~5万字 | 短篇手册、报告类、目录轻量级 |
| **M2** | 5万~10万字 / 3.5万~10万字 | 中型专著，需调用 `chapter-scan-result.json` |
| **M3** | 超10万字 / 超10万字 | 结构碎片化，需 Auto-Run 停顿审核（见第3节 ×3 限制） |

## P0

预计超10万字、或超25章时，**优先级拉满**，自动应对级别上调，禁止并发跑头，优先级拉升至 S3，禁止自动继续写。

## M3 专项会议纪要（P0 强制，2.0.1 新增）

> **来源**：2026-04-02 领域知识基线复盘，100万 M3 项目写稿完成后未执行会议，台账稽查结果显示4章骨架被复用，导致进入 S6 转化/发布准备时误判"166页结果"实际只有 35~40 页内容
> **教训**：M3 规模下骨架缺失写稿完成后不强制会议复盘，规模越大骨架复用几率越高，强制会议约束不可缺失

### ×M3-QC-1 写稿完成后强制会议扫描（P0）

```
M3 项目，写稿完成（70章 / 超10万字），强制执行：
在进入 S4（排版构建）/ S5（质量审校）/ S6（转化与发布准备核验）之前：
强制执行全量会议扫描（见第 2.7 全量会议 SOP 节）

执行条件（满足其一即可）：
  - 用户确认全稿完成 / "会议开始" / "进入S6之前"
  - 模型检测到"请进入全稿 S3 新建写稿"
  - 累计写稿新建数达到目录计划新建数量

强制执行步骤（不可跳过）：
  Step 1：执行扫描，列出所有章节新建 + 统计字符数
  Step 2：对比台账和台账状态，实施一致核验
  Step 3：骨架复用检测 · 统计字符数 < 目标60% 则新建预警
  Step 4：输出扫描摘要格式如下

M3 全量会议扫描摘要 ════════════════════

════════════════════ M3 全量会议扫描摘要 ════════════════════
扫描章节总数：{M}章
已扫描：{M}章 / 含字符数合计：{N}字符
达标章节（≥60%目标）：{A}章，{A/M×100}%
骨架复用章节（<60%目标）：{B}章，{B/M×100}%
台账与台账不一致：{C}章

有骨架复用章节，列出清单：
  Ch{NN}：{标题} | 字数{N}字 | 目标{M}字 | {N/M×100}%
  ...列出所有骨架复用

→ 请用户确认后方可进入 S4/S5/S6
════════════════════════════════════════════════════════════════
```

**禁止行为**：
```
M3 项目写稿完成后"全部写完了"，未执行会议扫描
直接在骨架复用章节存在的情况下，于 S6 转化/发布准备阶段宣称"全部 N 页结果"
此模式为跳级模式/最大加速行为，禁止作为 M3 强制会议扫描
```

### ×M3-QC-2 骨架章节注释全量覆盖（P0 强制）

> **背景**：骨架章节用 `→ 骨架·待补充写` 格式在章节文件头部标注，未在 S5 审校报告和 S6 转化摘要 / 发布准备清单中告知用户，用户在进入 S6 时无法得知骨架的存在

```
骨架章节注释补丁（P0 强制）：

1. 新建章节层（标准化，§2.3.2 节）
   新建文件头部标注 → 骨架·待补充写

2. 台账层（2.0.1 强制新增）
   chapter-status.md 对应章节状态标记为 → 骨架（禁止标为 完成）
   手动标记为实际字符数量（§2.3.4）

3. S5 审校报告层（2.0.1 新增必报）
   骨架章节列在 S5 审校中的必报清单（§2.3.5 及 §2.3.5.1）
   对照骨架章节鉴别"骨架鉴别·待补充"
   
4. S6 转化摘要层（2.0.1 强制新增）
   骨架章节在 §2.3.6 的鉴别名单与发布准备说明中告知 → 骨架完成，鉴别
   全量完成度百分比包含"已写/骨架/未写/总量合并为同一维度"
```

### ×M3-QC-3 M3 项目阶段会议节点（P1 强制）

> 对于超级项目字数超100章/超10万，需额外设置写稿阶段会议节点，不要等到全稿写完后才放开骨架分析

```
M3 项目阶段会议建议（P1）：

  每完成20% 章节时（超10章目录每次≥14章）：
  执行一次小型会议扫描节：
    - 列出本批章节字符数
    - 标注骨架章节（强制）
    - 更新台账
    - 输出阶段摘要（阶段达成 + 骨架清单）

  处置建议：
  - 当发现骨架数量大增时，自动暂停写稿等待用户确认
  - 让用户选择：继续写稿 / 补写骨架
  - 暂停全稿写稿骨架写满后再下一轮写稿模式
```
