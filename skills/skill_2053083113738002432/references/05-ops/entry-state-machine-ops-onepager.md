# 入口状态机（运营 1 页版）

> 版本：v1.1  
> 目标：给运营/交付同学一页可执行口径，统一「用户一句话进来后怎么走」。

---

## 一、先判三件事（每次必做）

1. **是否锚定书稿根目录（bookRoot）**  
   未锚定不得进入正文流程。
2. **是否有可用恢复卡（热恢复 / 半恢复 / 冷启动）**  
   先恢复，再执行，不重复全库扫描。
3. **是否命中高风险事实（价格/法规/版本/模型）**  
   命中即走风险门禁链。

---

## 二、8 个入口状态（运营口径）

- `E0` 项目锚定：先确认本次操作哪本书。
- `E1` 冷启动：无 `.fbs` 或无可用恢复工件。
- `E2` 热恢复：恢复卡完整且新鲜，直接续写。
- `E3` 半恢复：恢复信息过期/缺字段，先校准。
- `E4` 意图分流：写作 / 质检 / 交付 / 治理 / 退出。
- `E5` 风险门禁：离线降级、高风险事实、主体漂移。
- `E6` 执行链：按意图进入对应脚本链。
- `E7` 退出收口：保存恢复卡与变更摘要。

---

## 三、常见话术 → 处理动作

- **“福帮手 / 继续”**：优先热恢复；无恢复则冷启动最小集。
- **“写第X章 / 改写”**：先门禁再执行，不先直接改正文。
- **“质检 / 交付”**：优先审计链，失败先修再交付。
- **“搜最新 / 验证价格版本”**：强制双源 + 时间锚。
- **“退出”**：先保存会话，再结束。

---

## 四、红线（必须遵守）

- 未锚定 `bookRoot` 禁止进入正文执行。
- 高风险事实禁止单源结论直出。
- 治理工件禁止落在运行目录（统一 `.fbs/governance/`）。
- 声称“已完成/已通过”必须附可复核工件路径。

---

## 五、默认执行顺序（简化）

`锚定 -> 恢复 -> 分流 -> 风险门禁 -> 执行 -> 收口`

---

## 六、最小命令集（运营可记忆）

- 日常执行：`midterm:chain`
- 审计收口：`run-p0-audits`
- 治理汇总：`midterm:governance` / `midterm:milestone`
- 漂移修复：`governance:normalize`
