# 联网检索：反向验证查询（模型 / 价格 / 版本）

> **总纲**：与「知识截止补强、时效锚定、时态验证、方法论增强」四支柱对齐，见 [`web-search-strategy-deep.md`](./web-search-strategy-deep.md)。

> WorkBuddy 复盘 2026-04-16：用**旧模型名**（如「GPT-4o API pricing」「Claude 3.5 Sonnet API pricing」）作为主搜索词，会在结果页上**锚定旧条目**，同页若出现新版本（如 GPT-5.4、Sonnet 4.6），容易被忽略。

## 强制流程（涉及价格 / 模型 / 版本更新时）

1. **先确认「谁是最新」**  
   搜索：`[厂商] 最新模型 2026` 或 `GPT-5.4 发布 2026` / `Claude Sonnet 最新版本 API`（以当前年为准）  
   目标：从摘要中**抽取**官方或权威来源给出的**当前主力模型名称**。

2. **再以最新名称为锚查价格与规格**  
   例如：`OpenAI GPT-5.4 API pricing`（在第一步确认名称后）。

3. **禁止**  
   仅用已知旧模型名（GPT-4o、Claude 3.5 等）作为**唯一**主词去查「最新价格」。

## 同页多版本陷阱

若页面「Last updated」很新，但文中并列多代模型价格：

- 必须核对**每一段价格对应的模型名**是否与当前要写的「最新」一致。
- 对外结论须引用**与模型名同一行**的字段，避免混用上下段落。

## 与 fetch / 工具配合

向 `web_fetch` 或宿主检索提问时，显式包含：

- 「当前最新旗舰模型是哪一个？」
- 「本页是否同时出现多代模型价格？请分列。」
