# 离线版使用与升级到在线增强版指南

> **版本**：3.0.0
> **适用对象**：准备先离线使用，再联网升级增强能力的用户  
> **定位**：上架包内的权威入口，统一说明“离线基础版能做什么、在线增强版多了什么、如何升级”。

---

## 一、先看结论

| 模式 | 现在能做什么 | 当前不能做什么 | 如何进入 |
|------|-------------|---------------|---------|
| **离线基础版** | `general` 通用规范写作、本地素材分析、S/P/C/B 质检、Markdown/HTML 交付 | S0–S2 联网检索、会员/授权在线校验、远端场景包热更新 | 直接安装后即可用 |
| **在线增强版** | 在离线基础版之上，增加联网检索、会员/授权校验、增强场景包规则、规则热更新 | 仍遵守宿主能力与权限边界 | 联网后完成校验/激活 |

> **重要边界**：AI 会自动识别你想写的是家谱、白皮书、调查报告等体裁；但**自动识别 ≠ 一定已经启用增强规则**。未完成在线校验、门槛不足或远端不可达时，会**明确告知并回退 `general`**，不会静默伪装成增强版已生效。

---

## 二、离线基础版可直接使用的能力

离线时，以下能力默认可用：

- **通用长文档写作**：使用 `general` 通用规范完成书籍、手册、白皮书草稿等内容生产
- **本地素材盘点与重组**：围绕你已经提供的文档、笔记、章节草稿继续写作
- **S/P/C/B 质检**：本地规则可继续执行，不依赖联网
- **虚拟书房初始化**：创建 `.fbs/`、`deliverables/`、`releases/` 等本地工作面
- **Markdown / HTML 本地交付**：基础交付链路不依赖远端服务

离线时默认**不保证**的能力：

- **S0–S2 联网情报收集**
- **远端会员 / 企业授权校验**
- **增强场景包远端规则与热更新**
- **依赖在线校验的 premium 体裁增强规则**

---

## 三、从离线版升级到在线增强版

### 第 1 步：恢复网络后检查当前状态

```bash
node scripts/wecom/verify-member.mjs --check
```

这一步用于确认当前层级、乐包余额，以及是否已经具备进入在线增强版的条件。

### 第 2 步：如有企业授权码，完成激活

```bash
node scripts/wecom/verify-member.mjs --activate <激活码>
```

- 激活码格式示例：`FBS-XXXX-XXXX-XXXX-XXXX`
- 如果当前网络暂时不稳定，系统会先**本地暂存**，下次联网时自动完成二次验证

### 第 3 步：重新触发目标体裁

完成联网校验后，重新说出目标，例如：

- “写家谱”
- “写白皮书”
- “写调查报告”

系统会再次尝试加载对应场景包；若校验通过，将启用增强规则与热更新能力。

---

## 四、升级后会得到什么增强

联网并完成校验后，可逐步启用：

- **S0–S2 联网检索**：证据驱动写作、时间标签、来源补查
- **增强场景包规则**：针对 `genealogy`、`consultant`、`ghostwriter`、`training`、`personal-book`、`whitepaper`、`report` 的更强体裁规则
- **规则热更新**：远端规则版本刷新，不必重新安装整包
- **会员 / 企业授权能力**：基于乐包门槛或授权码启用更高阶体裁和配置

---

## 五、体裁识别、回退与升级之间的关系

| 场景 | 系统行为 |
|------|---------|
| 你明确说“写白皮书” | 先识别 `whitepaper` 为目标体裁 |
| 已联网且校验通过 | 启用 `whitepaper` 增强规则 |
| 未联网 / 未校验 / 门槛不足 | 明确告知后回退 `general` |
| 之后恢复网络并完成校验 | 再次触发该体裁即可升级为在线增强版 |

> **原则**：先保证可写、再逐步增强；任何回退都必须可感知，不能静默切换。
