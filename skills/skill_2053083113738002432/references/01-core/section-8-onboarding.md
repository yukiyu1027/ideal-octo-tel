## §8 新手引导体系

> **本章定位**：新手引导的交互与信息架构；具体组件由宿主实现

### 8.1 系统架构

#### 虚拟书房定位

> **新手第一动作**：进入福帮手后，先用日常语言说明当前工作区已就绪，再告诉用户「这套工作区在 FBS 里叫“虚拟书房”」，然后进入具体写作路径。

- **空间定义**：虚拟书房是当前项目的专属工作空间，不再等同于单一 `.fbs/` 目录；它由 `.fbs/` 内部台账层、`deliverables/` 对外交付层、`releases/` 发布准备层共同组成。
- **容器职责**：统一承载素材库、作品情报站、故事库、作者元知识、会话纪要、章节台账、S6 转化产物、S5 交付物、发布映射与反馈回流记录等核心资产。
- **界面职责**：`资产面板` 是虚拟书房的可见界面，负责让用户查看资产分区、状态变化、转化去向、发布进度与当前重点任务。
- **成长职责**：`乐包` 是虚拟书房的成长信号，用来反馈建书房、写章节、过质检、完书、转化、发布与回流修订等行为的积累与解锁，不再以孤立积分系统理解。
- **交互职责**：这里也是用户与创意会、读者会、对抗会、评审会等虚拟角色交流与留痕的空间；这些角色共同构成书房中的讨论室、辩论室与评审室。
- **转化职责**：当作品进入 S6 时，虚拟书房继续承担"成果转化层"的职责，把已完成内容转成传播单元、知识产品路线图、场景包来源与 AI 分身素材，并在需要时继续推进到发布区与反馈回流层。
- **执行映射**：初始化 `.fbs/` 内部台账层、`deliverables/` 对外交付层、`releases/` 发布准备层，并生成基础工件、准备资产索引等动作，统一称为**构建虚拟书房**，不再只以"部署工件"表述。


#### 三层架构



```
前端交互层：
  - S0.5引导
  - 偏好面板
  - 资产面板
  - 模式切换

后端服务层：
  - NLU拦截器 (NLUInterceptor)
  - 用户分层服务 (Profile)
  - 偏好管理API (Preference)
  - 资产管理API (Asset)
  - 自动化规则API (Automation)

本地持久层：
  - onboarding状态 / preference配置 / assets索引 / rules配置
```

#### 模块依赖关系

```
NLUInterceptor (意图拦截)
    ↓
ModeSwitcher (模式切换) ← UserClassifier (用户分层)
    ↓                          ↓
NoviceGuideHandler (引导处理)  PreferenceStore (偏好存储)
    ↓                          ↓
          PreferenceAPI (偏好API)
                ↓
          AssetIndex (资产索引)
                ↓
          RuleEngine (规则引擎)
```

### 8.2 S0.5 引导流程

> **与主规范对齐**：宿主若将写书类触发词的 **S0 前置调研** 合并为首响，须遵守 [`SKILL.md`](../../SKILL.md) **「技能加载后的行为约束」触发词首响**：避免仅追问「写什么主题」而不做 S0 说明；主题已在用户话术中时应直接推进调研/定位

> **ESM 桥接说明（v2.0.1 评审补充）**：S0.5 在 section-3-workflow.md 的 8 状态 ESM（IDLE/INTAKE/RESEARCH/PLAN/WRITE/REVIEW/WRITE_MORE/DELIVER）中位于 **INTAKE 状态内部**，是 INTAKE → RESEARCH 转换的可选前置步骤：新用户（无历史记录）或 180 天回归用户进入 INTAKE 时，**优先走 S0.5 两轮对话**，完成后再进入 RESEARCH（即 S0 前置调研）；老用户/专家模式直接从 INTAKE 跳转 RESEARCH，跳过 S0.5。**S0.5 不是独立 ESM 状态**，不产生新状态名称，状态宣告仍为 INTAKE

#### 触发条件

| 条件 | 触发逻辑 |
|------|----------|
| 新用户首次登录 | 用户ID无任何历史记录 |
| 180天未活跃回归 | lastActiveAt < (NOW - 180天) |
| 用户主动触发 | 说"重新引导"/"重新开始新手引导" |

#### 两轮对话流程

**第1轮：了解你想写什么（轻松开场，禁止出现内部术语）**
- 输入框：单行，最多200字符
- 引导意图：帮用户说出"想写什么 / 手头有没有材料"，不要求用户理解系统概念
- 示例提示措辞：「想写什么呢？随便说说就行——比如"想写一本理财入门书"、"有一堆访谈记录想整理成报告"、"想为公司写白皮书但不知道怎么开始"都可以。」
- 禁止出现：「虚拟书房」「路径A/B/C」「S0」「WP1」等系统术语

**第2轮：AI 主动说明当前工作区已就绪，然后直接开始**

> 福帮手根据第1轮感知到的主题和内容特征，以自然语言说明现在打算怎么做，然后直接进入——用户若无异议无需操作；如需调整，说一声即可。

```
示例输出（新用户，有主题无素材）：
「好，我帮你写一本理财入门书。
 我先把这次写作的工作区准备好——资料、进度和后续交付都会放在这里。
 接着我做一轮主题调研：看看同类书都在讲什么、目标读者是谁，然后帮你出一个初步目录。大概需要 10 分钟。
 现在开始，有中间结果会随时告诉你。想换个方向或轻量版本，随时说。」

示例输出（新用户，有素材）：
「好，你有现成的访谈记录——我先帮你把这些材料盘一遍，看看能支撑什么样的结构。
 材料整理完我们再决定大纲怎么走，不用你先想好。
 我这就开始，你也可以把材料告诉我。」
```




> **注意**：不得主动展示多选项菜单让用户点选；须先宣告推荐、再允许用户修改（对照 SKILL.md §5 禁止菜单式等待）

#### 跳过引导
- **跳过**：说「跳过」「跳过引导」「专家模式」「直接开始」（单字「跳过」即可触发，无需说全称）
- **重新走一遍**：说「重新引导」「重新开始新手引导」

#### 画像选择表

| 画像 | 图标 | 描述 | 推荐模式 |
|------|------|------|----------|
| 探索者 | 🔍 | 刚接触写作的新手小白 | 轻量/标准 |
| 写作者 | ✍️ | 有明确写作目标的进阶用户 | 标准/完整 |

### 8.4 书房资产面板

> **定位**：资产面板不是独立页面，而是虚拟书房的主界面。用户应能从这里看到"这间书房里有什么、现在推进到哪里、下一步该做什么"。

#### 面板分区

| 分区 | 作用 | 典型内容 |
|------|------|----------|
| 书稿区 | 查看正在写的主作品 | 章节台账、目录、当前进度 |
| 素材区 | 查看可复用原料 | 素材库、故事库、作品情报站、作者元知识 |
| 会话区 | 查看角色讨论与决策留痕 | 创意会、读者会、对抗会、评审会纪要 |
| 转化区 | 查看 S6 产物与去向 | 传播单元、产品路线图、场景包来源、AI 分身素材 |
| 发布区 | 查看对外可用、已外发、反馈中与回流修改中的成果 | 待发布条目、已发布条目、反馈记录、回流修改任务 |
| 成长区 | 查看书房成长与解锁状态 | 乐包余额、累计来源、已解锁体裁/能力 |

#### 资产分类

| 类型 | 图标 | 颜色标识 | 说明 |
|------|------|----------|------|
| 草稿 | 📋 | #F59E0B | 未完成的片段、卡片或章节 |
| 写作中 | 📝 | #6366F1 | 正在推进中的主作品或章节 |
| 待审 | 🧪 | #06B6D4 | 已写完、等待质检或评审的内容 |
| 已完成 | ✅ | #10B981 | 已完成且通过当前阶段要求的完整作品 |
| 转化中 | 🚀 | #8B5CF6 | 已进入 S6、正在转成传播或产品资产 |
| 发布中 | 🌍 | #14B8A6 | 已进入发布区，处于待发布 / 已发布 / 反馈中 / 回流修改之一 |
| 归档 | 📦 | #6B7280 | 已归档的历史项目 |

#### 面板动作
- **查看**：按书稿区 / 素材区 / 会话区 / 转化区 / 发布区 / 成长区切换
- **移动**：改变资产类型或所在分区
- **归档**：将已完成、已转化或已进入发布区闭环的项目移入历史层
- **删除**：批量删除资产（需二次确认）

#### 资产状态流转图

```text
素材区 / 会话区 / 作者元知识
          ↓
        草稿
          ↓
        写作中
          ↓
        待审
          ↓
        已完成
          ↓
        转化中（S6）
          ↓
  发布中（发布区四态）
    ├─ 待发布
    ├─ 已发布
    ├─ 反馈中
    └─ 回流修改 ─────→ 转化中 / 写作中
          ↓
        归档
```

#### 主状态流转规则

| 当前状态 | 进入条件 | 下一状态 | 说明 |
|----------|----------|----------|------|
| 草稿 | 已形成可推进内容 | 写作中 | 进入主写作链 |
| 写作中 | 当前章节/作品初稿完成 | 待审 | 等待 S/P/C/B 或角色评审 |
| 待审 | 通过当前阶段质量门禁 | 已完成 | 可进入完成区 |
| 已完成 | 进入 S6 转化 | 转化中 | 开始传播、产品化、场景包化、AI 分身化 |
| 转化中 | 明确可对外使用 | 发布中 | 进入发布区开始对外映射与追踪 |
| 发布中 | 完成一轮对外使用并沉淀反馈 | 归档 / 转化中 / 写作中 | 成熟成果可归档，需修改则回流 |

#### 发布区四态

| 发布子状态 | 进入条件 | 下一状态 | 说明 |
|------------|----------|----------|------|
| 待发布 | 已完成发布映射，但尚未真正外发 | 已发布 | 等待进入真实渠道 |
| 已发布 | 已在目标渠道中使用 | 反馈中 / 归档 | 开始收集真实读者或客户反馈 |
| 反馈中 | 已收到可执行反馈，正在分析 | 回流修改 / 归档 | 判断是继续放大还是回到书房修改 |
| 回流修改 | 反馈明确要求回改 | 待发布 / 转化中 / 写作中 | 轻度修改回到待发布，重度修改退回内部链路 |

#### 乐包在面板中的显示原则
- **显示位置**：乐包余额、累计获得原因与当前能力枝进度应在成长区可见
- **显示语义**：强调"书房成长记录"，而不是单纯"积分余额"
- **触发来源**：至少覆盖建书房、章节完成、质检通过、整书完成、S6 转化完成、进入发布区、消化外部反馈、完成回流修订等关键动作

#### 乐包事件命名

| 事件 | 命名 | 含义 |
|------|------|------|
| 建立虚拟书房 | 点灯 | 书房正式启用 |
| 完成一章 | 添卷 | 书房新增一卷内容 |
| 质量通过 | 校定 | 内容经审校后可入架 |
| 完成整书 | 成册 | 主作品完整成形 |
| 完成 S6 转化 | 出架 | 内容从书架走向转化区 |
| 对外发布 | 开门 | 书房成果开始面向外部使用 |
| 进入反馈闭环 | 迎评 | 书房开始接收真实外部回声 |
| 完成回流修订 | 回炉 | 书房把外部反馈重新炼成内部能力 |

### 8.5 虚拟角色机制

> **定位**：虚拟角色机制不是外挂插件，而是虚拟书房中的会议室系统。用户每次发起创意会、读者会、对抗会、评审会，都是在书房中开启一次有目标、有留痕的角色会话。

#### 角色空间映射

| 会议 | 书房空间 | 主要用途 |
|------|----------|----------|
| 创意会 | 讨论室 | 发散选题、寻找切口、生成方向 |
| 读者会 | 阅读室 | 模拟不同读者的理解与疑问 |
| 对抗会 | 辩论室 | 压测观点、识别漏洞、强化论证 |
| 评审会 | 评审室 | 审查结构、表达、质量与可交付性 |

#### 留痕原则
- **写入位置**：会话摘要应进入 `sessions-summary` 与对应 `sessions/` 目录
- **可见位置**：资产面板的会话区可查看每次会议的结论与待办
- **作用边界**：角色会话用于辅助决策，不替代作者最终判断
- **S6 延展**：进入 S6 后，角色机制应继续参与传播拆解、产品优先级排序与转化风险复核

### 8.7 模式切换机制


#### 三种模式对比

| 特性 | 新手模式 | 熟手模式 | 专家模式 |
|------|----------|----------|----------|
| S0.5引导 | ✅ 强制 | ⚠️ 可选 | ❌ 跳过 |
| 偏好管理面板 | ✅ | ✅ | ✅ |
| 书房资产面板 | ✅ | ✅ | ✅ |
| 自动化规则 | ✅ 预置4条 | ✅ 预置加自定义 | ✅ 完全自定义 |

| 核心写作流程 | 标准流程 | 标准流程 | 标准流程 |
| B层审计 | ✅ 完整保留 | ✅ 完整保留 | ✅ 完整保留 |

#### 切换规则

| 操作 | 行为 |
|------|------|
| 新手到专家 | 立即切换，后续无引导 |
| 专家到新手 | 提示重新开始引导，确认后触发S0.5 |
| 新手到熟手 | 立即切换，保留偏好配置 |
| 180天未活跃 | 提示是否切换模式，不强制 |

### 8.13 已知限制

| 限制 | 影响 | 说明 |
|------|------|------|
| localStorage存储容量限制（5MB） | 大量资产可能超出 | 超出后需手动清理旧会话数据 |
| 用户模板上限10个 | 高级用户可能不足 | 当前版本固定上限 |
| 自动化规则仅客户端执行 | 多端不同步 | 多设备需重新配置 |
| 无跨设备同步 | 用户换设备丢失数据 | 依赖手动导出/导入工件 |
