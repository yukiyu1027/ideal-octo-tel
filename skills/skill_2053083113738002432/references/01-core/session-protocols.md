# 会议机制规范（Session Protocols）

> **版本**：3.0.0
> **定位**：本文件规范 FBS-BookWriter v2.0.1 引入的四种会议机制——创意会、读者会、对抗会、评审会。  
> **核心设计理念**：多智能体的价值不在于并行速度，而在于**认知视角的真正隔离**——不同角色在不了解彼此推理过程的情况下独立给出判断，然后碰撞。  
> **防卡设计**：每种会议有时长上限，超时自动收束；人类决策节点有限选项，不开放式追问。  
> **触发方式**：见 `intake-and-routing.md` 第六节。用户无需手动操作，按体裁和节点自动激活；也可说触发词随时召集。

---

## 通用规则（所有会议共用）

### 会议时长上限（防卡顿）

| 会议 | 正常时长 | 上限 | 超时行为 |
|------|---------|------|---------|
| 创意会 | 10-15 分钟 | 20 分钟 | 协调者强制输出 2 个方向，人类立即选 |
| 路径C轻量创意会 | 5 分钟 | 10 分钟 | 强制输出 1 个推荐方向，人类确认或否 |
| 读者会 | 5-10 分钟 | 15 分钟 | 强制输出关切清单，不等三方充分讨论 |
| 对抗会 | 10-15 分钟 | 20 分钟 | 综合者强制给出加固表述，人类二选一 |
| 评审会 | 15-20 分钟 | 30 分钟 | 强制输出 P0 问题清单，跳过 P1/P2 |

**人类决策节点不超时**：等待用户选择方向时不设时间限制，但 AI 只提供**有限选项**（2-3 个），不开放式等待。

### 会议简报（每次会议前必做）

召集任何会议前，**协调者**先产出会议简报（约500字），作为所有参会 Agent 的统一上下文输入。参会 Agent 只读简报，不直接读全部项目文件，避免重复 token 消耗和上下文污染。

```
━━ 会议简报 · {会议类型} ━━
书名：{书名}
主题：{一句话核心主张}
体裁与读者：{简述}
当前状态：{当前所在阶段及已完成的关键产出物}
本次目标：{这次会议要产出什么，一句话}
关键背景（≤300字）：{从大纲/情报/原料中提炼的最关键信息}
━━━━━━━━━━━━━━━━━━
```

### 发散 → 收束两阶段原则

所有会议均采用两阶段结构，防止"框架锁定效应"（太早收束导致可能性丧失）：

- **第一阶段（发散）**：各角色独立表达，不打断、不批评对方
- **第二阶段（收束）**：在发散结果基础上碰撞，找出最有力的综合方案

### 心理契约（评审类会议前必问）

**读者会、对抗会、评审会**开始前，AI 向用户确认：

> "这次会议会提出质疑和问题。你希望侧重什么？  
> ①「发现问题」——尽量多找毛病  
> ②「均衡」——问题和亮点并重  
> ③「确认亮点」——以鼓励为主，问题点到为止"

用户选择后，会议强度按选项调整。默认为②均衡。

> **PATH-P1-01修复说明**：原选项用 A/B/C 字母标注，与本 Skill 的「路径A/B/C」（写作路径）字母体系冲突，已改为①②③避免歧义。

### 会议纪要格式（所有会议共用尾部格式）

```
━━ 会议纪要 · {会议类型} · {日期} ━━
核心结论：{一句话}
关键决策：{列出 1-3 条}
被放弃的方向：{列出，附原因}
待人类确认的问题：{列出}
下一步：{立即要做的事}
━━━━━━━━━━━━━━━━━━
```

自动追加至 `.fbs/sessions-summary.md`（只写纪要尾部，不写全程记录）。

## 创意会（Creative Session）

### 定位

在大纲方向锁定**之前**，产出 2-3 个差异化方向供人类选择或融合。  
创意会的产出是**方向提案**，不是方向锁定。允许写到一半重新召集（用触发词"重开创意会"）。

### 参会角色（3 个）

| 角色 | 立场 | 核心任务 |
|------|------|---------|
| **激进派** | 打破常规 | 如果完全颠覆读者预期，这本书可以怎么写？ |
| **守序派** | 读者最熟悉 | 按读者最熟悉的逻辑，最稳妥的结构是什么？ |
| **读者派** | 读者视角 | 作为目标读者，我最想先看到什么？最怕看到什么？ |

### 执行流程

```
【第一阶段：发散（各自独立，不看对方）】

  激进派产出：
    · 开篇策略（从什么地方切入？）
    · 章节逻辑（内容按什么顺序展开？）
    · 风格基调（和读者保持什么关系？）
    · 最大风险（这个方向最可能失败在哪？）

  守序派产出：（同格式）

  读者派产出：（同格式）

【第二阶段：碰撞（三方互相质疑）】

  规则：每方指出其他两方的最大弱点（各一句话），不重复对方自己说的风险。

【第三阶段：综合（协调者输出）】

  协调者梳理出 2-3 个可执行方向：
  · 方向1：基于[激进/守序/读者派]的核心逻辑
  · 方向2：融合[X]和[Y]的混合路线
  · （可选）方向3：另一个角度

【人类决策】

  "你倾向哪个方向？或者告诉我你想融合哪些元素。"
  → 人类确认后写入 .fbs/sessions/creative-session.md，进入 PLAN 状态
```

### 路径 C 轻量版（主题未锁定时使用）

在 10 分钟内完成，三方各说一句话（方向 + 理由），协调者给出 2 个主题/体裁的组合建议，人类选一个后进入路径 A 或 B。

### 产出物

`.fbs/sessions/creative-session.md`：完整会议记录 + 人类选定方向 + 放弃方向及原因

## 读者会（Reader Session）

### 定位

在 S2 大纲确认后，从真实读者视角检验大纲，找出结构性盲点和未被回应的期待。  
**核心原则**：读者会的根基是**真实数据**（L5 读者语言感知采集的真实讨论），不只是角色扮演。若 `reader-language.md` 已存在，会议开始前必须加载其中的「关切清单」和「词汇禁忌清单」作为读者 Agent 的基础认知。

### 参会角色（3 个）

| 角色 | 画像 | 核心视角 |
|------|------|---------|
| **新手读者** | 对主题完全陌生 | 看完目录，我知道这本书要解决我什么问题吗？ |
| **有基础的读者** | 已了解基础，寻求进阶 | 这本书能给我带来我还不知道的东西吗？ |
| **挑剔的读者** | 看过很多同类书 | 这本书的独特性在哪里？我有理由不看同类书而看这本吗？ |

### 执行流程

```
【数据加载（会议前）】
  读取 .fbs/reader-language.md（如存在）
  · 加载真实读者用语作为参会 Agent 的语言基础
  · 加载词汇禁忌清单作为语言风险提示

【第一阶段：各读者独立提问】
  每个读者 Agent 提出 5 个疑问或期待（不重复）：
  格式：
    [疑问] {一句话描述这个读者的疑惑}
    [期待] {一句话描述这个读者最想看到什么}

【第二阶段：汇总】
  协调者去重合并，产出：
  · 「读者关切清单」：约10条，按重要性排序
  · 「语言风险提示」：目录/章节名中出现的词汇禁忌

【人类决策】
  AI 对每条关切给出建议：
  · 建议在大纲中回应（高优先级）
  · 可以在前言/序言中回应
  · 接受这个疑问存在（不是所有问题都要回答）
  
  人类确认后，更新大纲，记录决策到 .fbs/sessions/reader-session.md
```

### 产出物

`.fbs/sessions/reader-session.md`：关切清单 + 人类决策（已回应/暂不回应/接受存在）  
`reader-language.md`：（已在 S0-L5 产生，此处只读取）

## 对抗会（Argumentation Test）

### 定位

对书中的**核心论断**进行压力测试，找出弱点并在书中提前回应，使论断更加严密。  
**触发时机**：书中出现新的核心论断时，AI 判断是否需要压力测试（用户也可随时触发）。一本书通常有 1-3 个核心论断值得做对抗会。

### 参会角色（3 个）

| 角色 | 立场 | 核心任务 |
|------|------|---------|
| **支持者** | 论断的辩护人 | 这个论断最有力的 3 个支撑论据是什么？ |
| **反对者** | 论断的挑战者 | 这个论断最容易被攻击的 3 个角度是什么？ |
| **综合者** | 仲裁者 | 在支持和反对之间，找到可以接受的最强表述 |

### 执行流程

```
【论断声明】
  AI 或用户说出要测试的论断（一句话，≤50字）

【第一阶段：支持者出场】
  · 最有力的论据1（含来源或逻辑链）
  · 最有力的论据2
  · 最有力的论据3
  · 这个论断在什么条件下成立？（边界条件）

【第二阶段：反对者出场】
  · 最容易被攻击的角度1（并说明攻击方式）
  · 最容易被攻击的角度2
  · 最容易被攻击的角度3
  · 这个论断在什么情况下会失效？

【第三阶段：综合者出场】
  · 保留论断核心，修改表述以回应最严重的反驳
  · 提出"预防性回应"：在书中哪个位置，用什么方式提前承认这个弱点并说明为何仍然成立
  · 产出"加固后的最强表述"

【人类决策】
  接受加固后的表述？还是修改论断？还是接受原论断的弱点（并注明风险）？
```

### 产出物

`.fbs/sessions/adversarial-session.md`：论断原文 + 三方论点 + 加固后的最强表述 + 人类决策

## 评审会（Review Session）

### 定位

在 S5 终审前，对完整书稿做结构性质疑，找出单 Agent 自审中容易放过的问题。  
**核心价值**：写书的 Agent 和审书的 Agent 认知视角真正隔离——审稿 Agent 不知道写书时的思路，不会受写稿心理定势影响。

### 参会角色（3 个）

| 角色 | 专长 | 核心问题 |
|------|------|---------|
| **内容专家** | 论点与论据 | 论点是否成立？有没有关键论据缺失？哪里有逻辑跳跃？ |
| **表达专家** | 叙事与节奏 | 哪些章节节奏拖沓？哪里读者会失去耐心？哪里太突兀？ |
| **怀疑论者** | 读者的批评视角 | 这本书最容易被批评的地方？读者最可能反驳什么？ |

### 执行流程

```
【心理契约（必做）】
  见通用规则第三节
  
【数据加载】
  · 加载 .fbs/sessions/reader-session.md（已承诺回应的读者关切）
  · 加载 .fbs/sessions/adversarial-session.md（已加固的核心论断）
  · 这两份文件中已解决的问题，评审会不重复提出

【第一阶段：三方独立审查（各负责全书）】
  内容专家输出：
    · 论点清单（列出全书核心论点）
    · 论据缺口（哪些论点缺乏足够支撑？）
    · 逻辑跳跃（哪里推理不严密？）
  
  表达专家输出：
    · 节奏问题（哪章太长/太短？哪里密度失调？）
    · 失去耐心的风险点（具体段落或章节）
    · 开头/结尾评估
  
  怀疑论者输出：
    · 最容易遭到批评的 3 个点
    · 读者最可能反驳的论断
    · 与同类书相比的弱势之处

【第二阶段：问题清单整理】
  协调者去重合并，按严重程度分级：
  · P0 必须修改（影响核心论点或可信度）
  · P1 强烈建议修改（影响读者体验）
  · P2 可选优化（锦上添花）

【人类决策】
  对每条问题：修改 / 接受风险（注明原因） / 留待下一版
```

### 产出物

`.fbs/sessions/review-session.md`：分级问题清单 + 人类决策（修改/接受风险/留待下版）

## 快速参考：四种会议对比

| 维度 | 创意会 | 读者会 | 对抗会 | 评审会 |
|------|--------|--------|--------|--------|
| **节点** | S1前 | S2后 | 核心论断出现时 | S5前 |
| **A级触发** | 路径C必须；路径B的A级必须；路径A可选 | 必须 | 按需 | 必须 |
| **B级触发** | 可选 | 可选 | 按需 | 可选 |
| **角色数** | 3+协调者 | 3+协调者 | 3 | 3+协调者 |
| **核心产出** | 方向提案 | 关切清单 | 加固论断 | 分级问题清单 |
| **人类决策** | 选方向 | 选择回应策略 | 接受/修改论断 | 修改/接受风险 |
| **可重复召集** | 是 | 是 | 是 | 是 |
| **触发词** | "开创意会" | "开读者会" | "压力测试这个论点" | "开评审会" |
