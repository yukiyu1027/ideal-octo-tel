### 原料充分性检查（S2.5 会议 P0 强制）
> **来源**：以往项目经常在 S3 写稿时才发现原料不足，导致要回退到 S3 写稿阶段，影响写稿质量不连贯。
**充分性检查标准**：见 `.fbs/material-library.md`，
- 已录入原料：{N}条，细分 {A}篇文章 / 合计 {D}字 / 访谈 {Q}条 / 其他 {O}条
- 联网查证记录：{S}条，分布在 `.fbs/search-ledger.jsonl` kind=material
- 是否达标：**参考 `search-policy.json` 的 `materialSufficiency` 门槛判断（单位统一为素材条数 `items`）**
  - 规则 5~10万字：推荐原料≥30条；少于10条判定严重不足
  - 规则 10~30万字：推荐原料≥80条；少于25条判定严重不足
  - 规则 30~100万字：推荐原料≥200条；少于60条判定严重不足
  - 规则 >100万字：推荐原料≥500条；少于150条判定严重不足（100字以上才算有效原料）

- **充分性不足的处置**：原料不足时，向用户说明缺口，建议补充，或确认接受以现有原料推进写作，不得跳过此步骤
**原料充分性核查表格**：

|---------|---------|---------|---------|------|---------|
| Ch01~3 | {数量} | {数量} | {N}条 | {状态} | {充足/待补充/原料类型} |
| … | … | … | … | … | … |

**待查找清单**：→ S3 写稿前
- [ ] {待查找原料1} 来源建议：{关键词 query / 推荐数据库 / 可搜索平台与来源}
