## 阶段 5：质量审校

**参考** ：`quality-check.md` 第1节 · **C0** · 全量审校 SOP

### §3.5 S5 审校流程

```
审校流程：
  调用 Proofer（审校角色），启动全量审校 + Mermaid图 + 脚注 + 注释一致性
  调用 Proofer（语言层审校，拼写检查）+ 错别字检查 + 过渡段 + 去AI味
  调用 Researcher（原料回溯审核，确认来源引用是否完整）
  分级审校·按章节重要性分 S/P/C/B 四级
  调用 CX（跨章一致性审核），见 cross-chapter-consistency.md
  调用 C0-1（全书去AI味扫描）→ 确认通过后进入 S6，见 book-level-consistency.md
  调用 C0-2（重复内容 / 冗余表述检查）→ S1、S2 定稿后的二次检查
  输出质量审校报告（综合打分）
  调用 Proofer（最终核查）+ B 层评分（B2）+ 输出审校通过通知
  完成 **S5 审校结束**
  等待 用户确认
  等待 确认进入 S6
  等待 确认继续
```

### §3.5.1 S5 数据新鲜度必报规则（P0 强制）
> **说明**：数据密集型书稿（含统计数据/调研数据/事件时间线），S5 审校必须覆盖数据新鲜度，不得跳过。

**数据新鲜度必报（P0 强制）**：
```
S5 审校结束时，强制输出数据新鲜度表（P0 强制）：

════════════════ → S5 审校 ════════════════
数据新鲜度总表（{M}章，含数据引用章节：{N}章）
数据截止时间：{ceil(M÷0.2)}章，达标（80%以内）：{N}章，达标率：{N/M×100}%
数据新鲜度分布（高时效性风险）：

  各章需注意：  时效风险章节清单（需标注更新）：
  如N < ceil(M÷0.2)，表示有不足20% 章节  存在过期数据风险，建议检查最新数据：read_file 重新确认数据出处
```

**数据新鲜度章节格式模板**：
```
数据新鲜度：[S3-ChNN] 章N.md | 数据来源：{数据来源名称} | 数据时效：{数据截止时间/标注来源}
```

**数据时效性分级阈值**：
| 时效级别 | 适用场景 | 数据新鲜度要求 |
|-----------|------------|
| 低时效 | 历史类 | 历史数据不要求100% |
| 中时效（1~5年） | 通用商业/科普 → 数据年份不超过5年 |
| 高时效（1~3年） | 11~20章，数据年份不超过3年 |
| 极高时效 | 21~30章，不超过 ceil(M÷0.2) 章 |
| 最高时效 | 超31章 | 超0章内时效风险 → 全量1章 |

> **时效说明**：超时效风险章节1章后，推荐重新确认来源→ 1次数据声明

### 审校角色编排

```
审校：
  角色 Proofer（语言层审校）：  调用语言审校 + Mermaid图 + 注脚 + 注释模板
  角色 Proofer（拼写语言审校）：  调用拼写检查 + 过渡段检查 + 去AI味审核
  角色 Researcher（原料回溯）：  调用原料来源回溯（确认引用合规来源）
  分级审校级别细分 S/P/C/B 分级，按分级安排重点审校 S5 审校时间
  角色 CX（跨章一致性）：  调用跨章一致性 cross-chapter-consistency.md
  角色 C0-1（全书去AI味）→ 确认通过后进入 S6，见 book-level-consistency.md
  角色 C0-2（重复/冗余内容）→ S1·CS2 定稿后二次检查
  输出质量报告综合打分
  角色 Proofer（最终核查）+ B 层审校（B2）+ 审校通过通知
  完成 **S5 审校结束**
  等待 用户确认
  等待 确认进入 S6
```
