# 内容模板体系

> FBS-BookWriter 参考文档 | 四种主结构 × 各章节类型的完整 Markdown 模板

---

## 一、前言模板（四层递进，所有结构共用）

```markdown
# 前言

## [破冰标题]

[具体人物名]，[年龄/身份]，[具体背景 2-3 句]。

[困境描述 2-3 句，带细节让读者代入]。

[转折点：某个事件/发现]。

[结果：量化的改变]。

## 这本书是什么？

**这本书是[一句话定位]。**

这本书**不是**[排除歧义 1]，也**不是**[排除歧义 2]。
[与竞品扫描的差异化建议注入此处]

你可以把它当作[使用方式]，[具体使用场景]。

## 你会在这本书中找到

- **第 1-3 章**：[认知/问题/总论 概述]
- **第 4 章**：[核心价值章节]
- **第 5-6 章**：[工具/案例/方法 概述]
- **附录**：[速查表/保鲜表/清单]

> **AI 辅助声明**：本书在资料整理、数据分析和初稿生成过程中使用了协作工具协同平台。
> 所有内容均经过协作工具审核、事实核查和独立判断。最终文责由作者承担。
```

**去AI味要求**：开场人物有名字、有年龄或身份、有具体数字。「某位企业家」不合格。

## 二、结构一：场景库型

### 认知章模板（单章 3500-5000 字）

```markdown
# 第 X 章 [章标题]

[题引 1-2 句概括核心问题，自动标记为 epigraph]

## X.1 [从故事/场景切入]

[案例：具体人物的困境+转折，500-800 字]

**[金句]**

## X.2 [核心概念/框架]

[零术语版定义，200-300 字]

| 维度 | 旧认知 | 新认知 |
|------|--------|--------|
| ... | ... | ... |

## X.3 [深化：能/不能]

- **好消息**：[机遇面]
- **坏消息**：[风险面]
- **关键变量**：[决定因素]

## 本章小结

[3-5 条 bullet + 1 个表格]

*下一章我们来看[下章核心问题]。*
```

### 工具章模板（单章 3500-5000 字）

```markdown
# 第 X 章 [工具/方法名称]

[题引]

## X.1 [痛点陈述，300-400 字]

## X.2 [工具定义：零术语版，200-300 字]

## X.3 [3-5 个使用场景，每个 200-400 字]

## X.4 [完整案例：背景→问题→过程→量化结果，600-1000 字]

## X.5 [实操清单，3-5 步]

## 本章小结

| 要点 | 说明 |
|------|------|

*下一章[过渡]。*
```

### 场景库模板

**单个场景（300-600 字）**：

```markdown
### [编号] [场景名称]

**痛点**：[2-3 句]
**解法**：[2-3 句]
**切入步骤**：
1. [可立即执行]
2. [第二步]
3. [第三步]

**预期效果**：
- 效率：从 X 小时 → Y 分钟
- 收益：X,XXX-X,XXX 元/月（基于公开数据推算，仅供参考）
  ^[来源：[名称](https://example.com)，检索日期：YYYY-MM-DD]

**适合人群**：[一句话]
**难度**：★ / ★★ / ★★★
```

**场景库整章组合**：

```markdown
# 第 X 章 [N 个场景实操]

[使用说明：如何用场景，200-300 字]

## 类别一：[类别名]（场景 1-N）
[类别概述 100-200 字]

### 场景 1 ...
### 场景 2 ...

## 类别二：[类别名]
...
```

## 五、结构四：阶梯递进型

### 单层级章模板

```markdown

[题引：说明本层级适合什么水平的读者]

## 前提检查

- [ ] 已掌握 L[N-1] 的 [具体能力]
- [ ] 已完成 [具体前置动作]

> 如果以上未满足，建议先回到 L[N-1]。

## [N].1 [核心技术/概念]

[教学内容，按由浅入深排列]

## [N].2 [实操练习]

**练习 1**：[具体任务描述]
- 预期耗时：X 分钟
- 预期产出：[具体产出物]

**练习 2**：...

## [N].3 [进阶挑战]

[超出本层级标准的延伸任务]

## 本层级小结

| 你学会了 | 你能做到 | 下一站 |
|----------|----------|--------|
| ... | ... | 进入 L[N+1] |
```

## 六、结语模板（所有结构共用）

```markdown
# 结语

[回扣前言故事，1-3 句呼应开篇人物的困境]

[时代意义：为什么此刻行动最重要，2-3 段]

[行动号召：具体的第一步，1-2 段]

**[全书总金句]**
```

**去AI味要求**：最后一句必须是具体行动，不要「AI将带来更美好的未来」。

## 八、Mermaid 图表体系

> 6类Mermaid图表模板已创建于 `mermaid-templates/` 目录
> 写作时直接嵌入 Mermaid 代码块，如同嵌入表格。配色注入规范见 `visual.md` §3。

### 决策树：何种内容配何种图表

| 内容场景 | 推荐图表 | Mermaid 类型 | 示例章节 | 模板文件 |
|---------|---------|-------------|---------|---------|
| 流程/步骤/操作 | Mermaid 流程图 | `flowchart TD` | 工具章·实操步骤 | `mermaid-templates/flowchart.md` |
| 时序/对话/交互 | Mermaid 时序图 | `sequenceDiagram` | 案例章·系统交互 | `mermaid-templates/sequence.md` |
| 状态转换/阶段 | Mermaid 状态图 | `stateDiagram-v2` | 框架章·生命周期 | `mermaid-templates/state.md` |
| 概念发散/分类 | Mermaid 思维导图 | `mindmap` | 认知章·知识结构 | `mermaid-templates/mindmap.md` |
| 阶段时间线 | Mermaid 时间线 | `timeline` | 案例章·实施阶段 | `mermaid-templates/timeline.md` |
| 项目排期/计划 | Mermaid 甘特图 | `gantt` | 行动清单·排期 | `mermaid-templates/gantt.md` |
| 数据对比/属性 | 表格（现有） | - | 通用 | - |
| 层级/递进/简单流程 | 流程块（现有 H4） | `↓├└→` | 通用 | - |

**选择原则**：有3个以上节点的流程/关系 → Mermaid；2个节点的简单对比 → 表格；纯线性步骤 → 流程块。

### 六类语法骨架模板

> **按需加载**：以上 6 个模板文件已外置，按图表类型按需引用
> 配色注入：`[book.color]`运行时替换为实际配色值
> 节点标签约束：≤15汉字，超长用`<br/>`换行（见 `visual.md` §3）

| 图表类型 | 模板文件 | 包含内容 |
|---------|---------|---------|
| 流程图 | `mermaid-templates/flowchart.md` | 基础模板/分支流程/并行流程/使用指南 |
| 时序图 | `mermaid-templates/sequence.md` | 基础模板/多方交互/自调用循环/使用指南 |
| 状态图 | `mermaid-templates/state.md` | 基础模板/多状态分支/状态与动作/使用指南 |
| 思维导图 | `mermaid-templates/mindmap.md` | 基础模板/多层级导图/使用指南 |
| 时间线 | `mermaid-templates/timeline.md` | 基础模板/详细时间线/使用指南 |
| 甘特图 | `mermaid-templates/gantt.md` | 基础模板/多阶段甘特图/使用指南 |

### 视觉密度规则

| 级别 | 阈值 | 说明 |
|------|------|------|
| **底线** | ≤ 5000 字 ≥ 1 个视觉元素 | Mermaid / 表格 / 流程块 / 插图标记均算 |
| **推荐** | ≤ 3000 字 ≥ 1 个 | 信息密集型章节（工具/框架章）建议此密度 |
| **上限** | ≤ 1500 字不超过 1 个 | 避免图多文少、阅读碎片化 |

> 密度检查由 S 层自动预过滤执行（V1 检查项），见 `quality-check.md` §1。
