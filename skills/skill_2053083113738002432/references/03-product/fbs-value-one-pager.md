> **版本**：2.1.1  
> **受众**：写作者、主编、采购与培训试讲；与 [`SKILL.md`](../../SKILL.md) 产品描述一致，无技术实现细节。

---

## 是什么

**福帮手（FBS-BookWriter）**是一套面向 **3 万字以上长文档** 的 AI 写作与交付工具链：从调研与大纲，到章节写作、分层质检，再到 MD/HTML 等格式交付。适合书籍、手册、白皮书、行业指南、长篇报道与深度专题。

---

## 你能得到什么

| 能力 | 说明 |
|------|------|
| **全流程** | 覆盖 S0–S6（从情报到转化/发布准备），按阶段推进，减少「写到一半不知道下一步」。 |
| **分层质检** | S/P/C/B 四层规则 + 综合评分，支持「去 AI 味」与自检，偏**成就式反馈**而非纯挑错。 |
| **长文档纪律** | 串行改稿、可见性、跨会话恢复（恢复卡、章节台账），适合长篇协作。 |
| **场景包** | 8 类垂直场景（通用内置 + 7 类增强场景）；不满足条件时**显式降级**，不静默假装。 |
| **联网与离线** | 宿主允许时可联网查证；**离线自动降级**，用已有素材与模型知识继续并标注。 |
| **交付** | 以 Markdown / HTML 为主，可对接构建与多格式导出（依赖宿主与可选插件）。 |

### 认知资产与「三化」

长文档在福帮手中的定位是**可持续运营的认知资产**，而不仅是单次稿件：

- **可进化**：阶段与台账落盘，改写与升级可追溯。
- **可分发**：同一书稿工程可按场景包与通道输出不同形态。
- **可增值**：与**场景包**、**乐包**、**离线/在线**分层能力协同，形成可升级的写作与交付路径。

详细口径与工程对齐见 [`references/05-ops/cognitive-asset-threeization.md`](../05-ops/cognitive-asset-threeization.md)。

## 怎么触发

在支持的宿主（如 WorkBuddy / CodeBuddy）中，用自然语言提到 **「福帮手」**、**「写书」**、**「写白皮书」**、**「定大纲」**、**「质量自检」** 等即可路由到本技能（完整触发词见 [`SKILL.md`](../../SKILL.md) 触发词段）。

---

## 你需要知道的两条边界

1. **首次进入书稿工作流**：应在书稿目录下按规范执行开场脚本（由 Agent 调用），以便刷新恢复信息与宿主能力；**仅把技能当文档读、不跑脚本**时，能力会明显受限。  
2. **退出会话**：说「退出 / 停止」时应先走会话退出脚本，再回复用户，**下次可从上次位置继续**。

## 与「离线版 / 在线增强版」

- 离线或弱网环境下仍可写作与质检；增强能力（部分场景包、在线校验等）可能受限。  
- 升级路径见 [`references/01-core/offline-online-upgrade-guide.md`](../01-core/offline-online-upgrade-guide.md)。  
- **乐包**为本地行为激励点数，用于解锁部分增值场景包门槛，详见 [`references/05-ops/credits-guide.md`](../05-ops/credits-guide.md)。
