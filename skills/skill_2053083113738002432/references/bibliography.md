# 参考文献索引（MAT 编号系统）

> **版本**：2.1.1
> **更新日期**：2026-04-11
> **用途**：统一管理所有引用资料的 MAT 编号


---

## MAT 编号规则

### 编号格式

- **格式**：`MAT-XXX`
- **XXX**：3位数字编号（001-999）
- **示例**：MAT-001, MAT-002, MAT-003

### 编号原则

1. **按引用顺序递增**：首次引用时分配 MAT-001，后续引用递增
2. **不可重用**：已分配的编号不可删除或重复使用
3. **保持连续性**：编号尽量连续，避免大跳跃（如 MAT-001 → MAT-003 → MAT-002）
4. **分类前缀**：可使用分类前缀（如 `REF-001`、`WEB-001`）区分来源类型

## 引用类型与格式

### A 级：网络资源

**格式**：
```
MAT-XXX | 来源 | 日期 | 访问日期
```

**示例**：
```
MAT-001 | GitHub | Octoverse 2025 | 2025-11-01 | https://github.blog | 2025-11-01
```

### B 级：报告/白皮书

**格式**：
```
MAT-XXX | 机构名 | 《报告名》 | 日期
```

**示例**：
```
MAT-002 | 中国信通院 | 《中国人工智能发展白皮书》 | 2025-10
```

### C 级：书籍/期刊

**格式**：
```
MAT-XXX | 作者 | 《书名》 | 出版社 | 年份
```

**示例**：
```
MAT-003 | 张三 | 《人工智能概论》 | 机械工业出版社 | 2024
```

### D 级：内部资料

**格式**：
```
MAT-XXX | 类型 | 标题 | 日期
```

**示例**：
```
MAT-004 | 内部文档 | 《项目需求文档 v1.0》 | 2025-11-01
```

## 引用使用规范

### 章内引用

**格式**：
```
见 MAT-XXX
```

**示例**：
```
根据 MAT-001 显示，全球 AI 市场规模达到 5000 亿美元。
```

### 章内引用（详细）

**格式**：
```
[来源描述]（MAT-XXX）
```

**示例**：
```
[中国信通院发布的《中国人工智能发展白皮书》]（MAT-002）
```

### 参考文献列表

每章末尾添加参考文献列表：

## 维护说明

### 新增引用

1. 检查参考文献列表是否已存在该引用
2. 如不存在，按顺序分配新的 MAT 编号
3. 更新本索引文件

### 更新引用

1. 如需修改引用信息，更新对应的 MAT 编号记录
2. 保留原编号，避免影响已有引用

### 删除引用

1. 标记为废弃（添加 `[已废弃]` 标签）
2. 不重新使用该编号

## 工具集成

### 自动化管理

建议实现 `scripts/bibliography-manager.mjs`：

```javascript
// 功能示例
- addEntry(type, source, date)      // 添加新引用
- updateEntry(matNumber, fields)       // 更新引用
- getNextNumber()                    // 获取下一个编号
- generateCitation(matNumber)         // 生成引用文本
- validateIndex()                    // 验证索引完整性
```

### 与文档同步

- 每次添加/更新引用时，同步更新 bibliography.md
- 提供批量导入/导出功能
- 支持 JSON 格式的机器可读版本

## 注意事项

1. **唯一性**：确保每个 MAT 编号对应唯一来源
2. **可追溯性**：记录分配日期，便于追溯
3. **准确性**：核实引用信息的准确性（作者、标题、年份等）
4. **完整性**：补充必要的元数据（URL、ISBN、DOI 等）
