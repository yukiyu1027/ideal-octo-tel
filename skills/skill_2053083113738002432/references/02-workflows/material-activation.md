# 素材整理 / 原料入口

## 1. 目标

解决“素材整理占比低、完成率低、容易无限 S0”的问题。

## 2. 何时进入

- 用户说“整理素材”“激活原料”“把这些文档先变成可写素材”。
- 用户上传多份资料，但还没准备直接写正文。

## 3. 完成信号

以下至少满足两项，才能判定“素材整理已完成”：

- 形成 `material-library.md` 或等价素材主仓。
- 明确缺口清单：还缺哪些资料。
- 明确可直接开写的章节或主题簇。
- 恢复卡中 `nextSuggested` 不再停留于“继续整理素材”。

## 4. 失败降级

- 资料不足：输出缺口清单，不强行卡在 S0。
- 用户只要尽快开写：给出“按当前资料直接起草”的明确选项。
- 如果素材库已达标，禁止无期限继续追问；参见 [`../01-core/s0-material-phase-guard.md`](../01-core/s0-material-phase-guard.md)。
