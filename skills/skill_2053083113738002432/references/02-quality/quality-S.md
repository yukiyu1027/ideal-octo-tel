## S层·句级（6条）｜逐句扫描

**正向目标：每句有信息增量，读者读完获得一个事实或画面**

---

### S1 首句具体性

**禁令**：禁用套话开场。任何章节、段落的第一句不得是抽象的趋势、概念、定义。

**正向**：首句必须含人物/场景/数据/问题之一。

**阈值**：全文首句违规数 = 0。

| 禁止 | 替代 |
|------|------|
| "在当今数字化转型浪潮中……" | 直接切入场景或问题 |
| "随着AI技术的飞速发展……" | 用一个具体人/事开头 |
| "众所周知……" | 删除，直接说内容 |

**正例**：`你花了30分钟按教程装好了。机器人上线了。然后呢？`
**反例**：`随着AI Agent技术的快速发展，越来越多企业开始关注如何将AI落地。`

**趋势性开篇（违规）**：以「市场/行业 + 快速增长/态势/浪潮」替代具体场景，违反 S1 禁止套话。
- ❌ `全球AI写作工具市场呈现快速增长趋势。`
- ✅ `上周你打开后台，发现导出队列里卡了 37 个任务。这才是你想解决的事。`

---

### S2 消灭冗余修饰

**禁令**：用数据替代程度副词。删掉修饰词后句意不变 = 该词多余。

**正向**：每个修饰词都携带不可替代的信息。

**阈值**：全文程度副词出现次数 = 0（见下方 A类禁用词表）。

| 原文 | 修正 |
|------|------|
| "非常重要的关键步骤" | "关键步骤" |
| "在很大程度上显著提升了效率" | "效率提升了X%" |
| "从根本上彻底改变了" | "改变了" |

---

### S3 短句优先

**禁令**：单句超40字（中文）必须拆分。

**正向**：句子节奏明快，信息密度高。

**阈值**：超40字句占比 < 10%。**P1 预警**：`quality-auditor.mjs` 在句数启发式 ≥ 2 且占比 **>8%** 时可配合 `--fail-on-long-sentence-warn` 非零退出（见 `scripts/quality-auditor.mjs`）。

| 原文 | 修正 |
|------|------|
| "通过将AI与企微工作流相结合，可实现从咨询到方案生成的全流程自动化" | "AI+企微工作流，全流程自动化。从咨询到方案生成，三步完成。" |

---

### S4 连接词降温

**禁令**：禁止高频使用过渡连接词做段间桥梁。

**正向**：段与段之间靠逻辑衔接，不靠连接词胶水。

**阈值**：同一连接词全文不超过3次；C类连接词总密度 < 2次/千字。

| 禁止（高频连接词） | 替代策略 |
|-------------------|---------|
| "此外/另外/同时" | 直接说下一个事实，不加过渡（**「同时」**列入 C 类密度检测词；单篇偶尔 1 次仍可能合规，但语义冗余时须删） |
| "值得注意的是/需要指出的是" | 删除，直接说内容 |
| "不仅如此/更重要的是" | 用短句另起，靠内容本身推进 |

**正例**：`效率提了3倍。成本？砍半。`（无连接词，靠节奏推进）
**反例**：`此外，成本也得到了显著降低。同时，用户体验也有所改善。`

---

### S5 buzzword黑名单

**禁令**：禁用中文科技圈AI高频术语。

**正向**：用读者能直觉理解的日常用语。

**阈值**：B类禁用词出现次数 = 0（见下方 B类禁用词表）。**机读计次**（易误报须人工）：`references/02-quality/s5-buzzword-lexicon.json` + `quality-auditor.mjs`。**`--enforce`** 已含；可单用 `--fail-on-s5-buzz`。

| buzzword | 替代 |
|----------|------|
| "赋能" | "让……能做……" |
| "抓手" | "切入点/工具" |
| "底座/底层能力" | "基础功能" |
| "链路/全链路" | "流程/从……到……" |
| "沉淀" | "积累/留下" |
| "闭环" | "从头到尾跑通" |
| "颗粒度" | "细致程度/精度" |
| "打通/拉通" | "连起来/接上" |
| "对齐/拉齐" | "统一/一致" |
| "心智" | "认知/印象" |

### S6 标点符号纪律

**禁令**：禁止标点模式单调化。AI特征是破折号滥用、分号堆砌、感叹号过度、标点类型单一。

**正向**：标点服务于节奏，类型多样，疏密有致。

**频率上限**：

| 标点 | 上限 | 说明 |
|------|------|------|
| 破折号（连续两个 U+2014） | ≤ 1次/千字 | AI最显著指纹，严控 |
| 分号（；） | ≤ 2次/千字 | 用句号断句替代 |
| 感叹号（！） | 全文 ≤ 2个 | 克制使用 |
| "具体来说"式冒号 | = 0 | 删除引导语，直接起列表 |

**二字横线（规范称破折号，机读计为连续两个 U+2014）的 AI 典型滥用模式**（实战复盘常见；须优先改句式，不单改标点）：

1. **横线重复括号内已说明的内容**：括号已解释，再加横线累赘。
   - ❌ `"观望"`+横线+`看看竞品动向` → ✅ `"观望"，看看竞品动向` 或合并为一句。
2. **句末横线接引出的独立主谓句**：把一句硬拆成「碎片 / 完整句」两截。
   - ❌ `渗透率……`+横线+`这是一个务实的路径` → ✅ `渗透率……这是一个务实的路径。`
3. **「选择+横线+意味着」「答案+横线+其实是」型**：用横线假衔接因果。
   - ❌ `选择`+横线+`意味着不被任何一个平台绑定` → ✅ `选择，意味着不被任何一个平台绑定` 或 `选择：意味着…`（择更顺的一种）。
4. **「本质上+横线」「换句话说+横线」型**：分析套路 + 横线引导。
   - ❌ `本质上`+横线+`这是一个入口` → ✅ `本质上：这是一个入口` 或删引导词。

**替换规则速查**（非穷尽，以读起来像人说的话为准）：

| 情形 | 倾向替换 | 说明 |
|------|----------|------|
| 解释已在括号/前文出现 | 逗号、句号或合并 | 避免「括号+横线」双解释 |
| 横线后是完整新句 | 前段用**句号**收束 | 忌横线后接「这……」「这意味着……」式续写 |
| 「答案/结论/选择」后强调 | **冒号** | 冒号直接带出宾语更干净 |
| 仅剩装饰性停顿 | 删横线或改逗号 | 删后若语义不变则属冗余 |

**密度快速估算**：全文检索连续 U+2014 出现次数计为 *N*；估算千字 *K*（正文字数÷1000）；密度 = *N*÷*K*（与「对」计数的口径与 C0-1 全书总账一致时，以工具/编辑器的「出现次数」为准）。**>1 次/千字** 须逐条对照上表或 [`quality-AI-scan.md`](./quality-AI-scan.md) 清单改写。

**交稿检查**：多路并行时建议按 [`quality-AI-scan.md`](./quality-AI-scan.md) 模板留痕；单 Agent 亦须至少完成横线密度计数与 S5/S6 对照。

**P0 执行门禁（与工具对齐）**：密度 **>** `search-policy.json` 的 `qualityGate.emDashPerThousandWarnAbove`（默认 **1/千字**）视为 **S6 未通过**，**不得**宣称本章 S 层已收口或进入 **S5 终稿**（除非用户书面接受风险）。可机读实现：`--fail-on-s6-warn`（仅 S6 警告带）；**`--enforce-strict`**（S6 含警告带 + 与 `--enforce` 同款 S/P/C/B、S5 B 类编号门禁）；**`--enforce`** 在 **S6 阻断**（密度 **>** `emDashPerThousandBlockAbove`）或 **S5 B 类词**（词表命中>0）等条件下失败，**不含** S6 警告带。密度 **>** `emDashPerThousandBlockAbove`（默认 3/千字）与 **`quality-auditor` / C0-1 脚本阻断**一致；**恰等于**该阈值属 **⚠️ 警告**（非 `--enforce` 的 S6 单独失败），与 `quality-check.md` C0-1 表一致。独立复测：`--dash-density`。

> **全书尺度**：上表阈值对 **章内扫描** 与 **全书合稿总账** 同时适用。单章均略低于阈值时，全书累加仍可能严重超标，须在 **S5 阶** 或 [`book-level-consistency.md`](./book-level-consistency.md) **C0-1** 做全书密度核算。

> **执行闭环**：Critic 报 S6 未达标后，Writer **须重写句式并复扫 S6**，禁止只做「横线机械改逗号」类无效替换（见 `quality-check.md` §1.0.1）。

**鼓励使用**：

| 标点 | 用法 | 示例 |
|------|------|------|
| 问号（？） | 反问/设问散布段中 | "续费率？87%。" |
| 省略号（……） | 留白、停顿、欲言又止 | "他犹豫了一下……然后把合同签了。" |
| 短句群（，。交替） | 制造节奏感 | "效率提了。成本砍了。人没换。" |

**AI特征**：`AI正在改变一切，从生产到销售，每个环节都在被重新定义。此外，效率也得到了显著提升；成本大幅降低；用户体验明显改善！`
**去AI味**：`AI改了什么？生产线。销售端。客服。但改得最狠的，是你的预期。`

## 禁用词速查表

### A类·程度副词（S2适用）

非常、显著、大幅、极大地、深刻地、全面地、前所未有、根本、彻底、高度、充分、深入、尤其、特别、相当、极其、格外、甚

**v2.1.1 新增·绝对化命令词（全书 ≤ 3 次）**：必须、一定、务必、绝不能、无论如何

> **机读词表真源**：`references/02-quality/s2-quality-machine-lexicon.json`（与 `s2-imperative-lexicon.json`、`quality-auditor.mjs`、`quick-scan.ps1` 同步；变更后须 `npm run audit:consistency`）。

> **新增词阈值说明**：「必须/务必/绝不能」等绝对化命令词在写作稿件（非规范文档/法律文书）中属 AI 风格强行注入，全书合计不超过 3 次（仅保留法律/安全红线场景）。超出时 S2 判❌，须改为建议语气或给出具体理由替代命令。
>
> **评分说明**：检出 A 类词时，当前脚本口径为 **S2 未通过**；建议在报告中显式列出命中词与位置，但不再叠加旧版"额外扣分"公式。

### B类·buzzword（S5适用）

赋能、抓手、底座、底层能力、链路、全链路、沉淀、闭环、颗粒度、打通、拉通、对齐、拉齐、心智、势能、卡位、卡点、组合拳、矩阵（非数学语境）、生态（非生物语境）、中台（非技术架构语境）、协同、联动、融合、整合、维度、视角、层面

> **说明**：「维度/视角/层面」在 **空洞套话**（无具体对象、连续堆砌）时按 B 类处理；正常技术语义（如「三维坐标维度」）不判违规。

### C类·过渡连接词（S4适用）

此外、另外、**同时**（密度检测词：易与口语「同时进行」混淆，稿内作过渡桥梁时按 C 类计次）、值得注意的是、需要指出的是、不仅如此、更重要的是、与此同时、除此之外、总的来说、综上所述、由此可见、不难看出、显而易见、再者、再者说、从另一个角度看、换个角度说

> **语义冗余**：密度未超阈值但段际可合并时，仍建议删连接词，靠事实推进（对齐待改进清单 P2-11）。
