# 全书级一致性（C0 · 汇编门禁）

> **定位**：C0 用于弥补“单章合规、全书漂移”的缺口，负责在合稿与交付前做全书尺度的一致性复核。  
> **边界**：C0 **不并入**当前 S/P/C/B 四层综合分；即使单章综合分达标，只要 C0 未通过，仍不得宣称终稿通过或进入对外交付。  
> **关联规范**：`quality-check.md`、`metrics.md`、`quality-S.md`、`quality-PLC.md`

---

## 1. 为什么必须有 C0

逐章质检可以发现句段层问题，但很难自动发现以下全书级风险：

- 同一术语跨章含义漂移
- 相同观点在多章重复展开，造成注水
- 章节各自达标，但合稿后全书节奏、标点或口径失衡
- 对外交付前的声明、统计口径、引用方式前后不一致

因此，C0 的任务不是“替代单章质检”，而是作为**全书汇编门禁**，在提交审核或发布前做最后一道总账核查。

---

## 2. C0 当前必查项

### C0-1 全书破折号（——）总密度

按计划纳入终稿的正文范围统计全书 `——` 密度，阈值与 `search-policy.json` 当前配置保持一致：

| 档位 | 条件 | 处理 |
|---|---|---|
| 通过 | `ρ_em <= 1/千字` | 记录数值即可 |
| 警告 | `1 < ρ_em <= 3/千字` | 列出超标章节并改写后复扫 |
| 阻断 | `ρ_em > 3/千字` | 不得宣称终稿通过 |

> 说明：该门禁与 `qualityGate.emDashPerThousandWarnAbove` / `qualityGate.emDashPerThousandBlockAbove` 对齐。

### C0-2 术语与缩写唯一含义

重点检查：

- 同一缩写是否在不同章节表示不同概念
- 核心术语是否出现多个近义替换却未声明
- 图表、正文、摘要、目录中的名词是否统一

若发现术语冲突，必须回写术语锁定记录并修正文稿，不能以“读者自己能猜到”为由放行。

### C0-3 重复、冲突与结构回声

重点检查：

- 重复段落、重复案例、重复结论
- 同一数据在不同章节数值不一致
- 前文给出立场，后文又无提示地反向表述
- 多章使用完全相同的展开模板，导致整书结构回声过重

### C0-4 交付前口径一致

提交审核前，必须确认以下对象口径一致：

- `quality-check.md` / `metrics.md` / `quality-gates-brief.md`
- 机读配置 `search-policy.json`
- 相关脚本输出与测试断言
- 打包产物中的副本与根目录源码

---

## 3. 执行闭环

1. **单章通过后**：完成常规 S/P/C/B 审校，但不得立刻宣称全书通过。
2. **进入合稿阶段**：由主编或汇编责任人执行 C0 总账检查。
3. **发现问题后**：回到对应章节改写，不允许只改表面标点而不改句式和结构。
4. **修订完成后**：对受影响范围重新执行单章质检，并重新跑 C0。
5. **全部通过后**：方可进入 S5 终审结论、打包与对外交付。

---

## 4. 建议输出

C0 复核结果建议至少输出以下内容：

- 本轮范围（卷 / 章节 / 文件数）
- 全书级风险 Top 列表
- `——` 密度结果
- 术语冲突清单
- 重复 / 冲突 / 回声结构清单
- 结论：`pass / warning / fail`
- 下一步动作：直接放行、修订后复扫，或阻断交付

---

## 5. 通过标准

满足以下条件时，C0 才可视为通过：

- 全书级阻断项为 0
- 关键术语、缩写、统计口径无未解决冲突
- 全书破折号密度未超过阻断阈值
- 文档、脚本、测试与打包产物不存在明显口径漂移

若任一项未满足，C0 结论应为 **不通过**，不得以单章平均分较高替代全书级门禁结论。
