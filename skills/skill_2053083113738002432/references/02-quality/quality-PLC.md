## §P P层·段级（4条）—— Critic-L1 逐段检测

**正向目标：每段回答一个具体问题，读者读完知道"所以呢"**

---

### P1 问题驱动

**禁令**：禁用「概念→特性→应用」的AI默认叙事结构。

**正向**：正确顺序是"问题→困境→工具"。

**阈值**：每节开头必须是问题或场景，不得是定义。**S3 审校节点**：Critic-L1 须显式勾选「本节首段是否定义驱动」；主编抽查时优先看各节首段。

| 概念驱动（AI味） | 问题驱动 |
|------------------|---------|
| "场景包是一种将AI能力标准化封装的方法……" | "客户问：机器人装好了，然后呢？这就是场景包要回答的。" |
| "知识集是企微提供的知识管理能力……" | "你有200页文档，客户问了个问题，你希望AI 30秒找到答案。知识集就干这个。" |

**反面案例库（典型违规开篇）**：以人物传记式年号定义起笔，无读者问题。
- 如 `郑樵（104-162年），字渔仲，南宋史学家……`（百科卡片）
- 如 `你想给家谱里那位「只闻其名」的祖先立一小传，第一步却不是文采，而是敢不敢写下「不详」。`

---

### P2 对话代替转述

**禁令**：禁止全篇转述无直接引语。

**正向**：用真实（或合理虚构的）对话拉近距离。

**阈值（强制）**：每 **2000 字 ≥ 1 处**直接引语（`「……」` / `"\u2026\u2026"` 等）。**未达标 = P2 未通过**，不得以「文风严肃」豁免。全书级抽查按汉字规模折算。

| 转述（AI味） | 对话 |
|-------------|------|
| "某位顾问表示AI对其业务产生了显著影响" | "老陈说，他当时心里咯噔了一下。" |
| "用户反馈该功能有效提升了工作效率"——**P2 违规** | "'以前写方案要两天，现在半天出初稿。'第一个试用客户的原话。" |

---

### P3 禁止对称排比

**禁令**：同段禁用「首先/其次/最后」或「不仅/而且/还」式平行结构。

**正向**：列举用bullet list或表格；叙述用节奏变化替代工整对仗。

**阈值**：全文对称排比 = 0处。

**量化补充（分支式罗列）**：同一章内，若 **`###` 级标题**命中「说法一/二/三」「阶段一/二/三」「路线一/二」「方案一/二」等 **平行分支骨架 ≥ 3**，视为 **P3 违规**（与 AI 模块化结构同罚）。`quality-auditor.mjs` 启发式已计入平行分支###过多章节。

| AI味 | 去AI味 |
|------|--------|
| "首先提高效率；其次降低成本；最后改善体验。" | "效率提了3倍。成本？砍半。续费率87%。" |

---

### P4 拒绝注水

**禁令**：删掉一段后上下文仍顺 = 该段是注水，删除。

**正向**：每段不可替代，删了就断裂。

**阈值**：注水段 = 0。

常见注水模式：
- 换说法重述（"换言之……""也就是说……"）
- 过渡句填充（"接下来我们来看……""下面将详细介绍……"）
- 万能拖尾："等等""以及其他方面"

**相邻小结（P2 建议升格核查）**：相邻两节末尾若均有「小结/总之/综上」类段落，**语义重叠 >30%**（人工对照或 n-gram 粗判）→ 删并为一处或改写为递进关系。

## §C C层·章级（C1–C5 参与层分）—— Critic-L2 逐章审查

> **计分说明（v2.1.1 修订）**：C1–C5 共 **5 条规则**换算 10 分：`通过条数 ÷ 5 × 10`；C5 章节衔接从「多路并行 P0 建议项」**升级为 P1 强制**，不区分串行/并行，纳入正式计分。

**正向目标：每章有观点、有立场、有节奏起伏**

---

### C1 承认局限 + 明确表态

**禁令**：禁止万能叙事（"AI将彻底改变一切"）和无立场对冲（"一方面……另一方面……"然后不下结论）。

**正向**：坦诚说不能，然后给出明确建议。

**阈值**：每章 ≥ 1次好消息/坏消息 + 明确表态。

模板：
```
**好消息**：[能做的事]。
**坏消息**：[不能做的事]。
**建议**：[明确建议/立场]。
```

**AI味**：`一方面AI提升了效率，另一方面也带来了新的挑战，企业需要综合考量。`
**去AI味**：`AI能帮你把200页文档变成可搜索的知识库。但它判断不了哪些信息对这个客户最重要，这需要你的行业经验。先上知识库，别急着上智能推荐。`

**章节小结（P0，结构完整必要条件）**：除终章/特别声明外，**每章须有 200–500 字的「本章小结」**（可独立 `### 本章小结` 或末节前一段），交代本章结论、局限与读者下一步。**缺失视同 C1 结构不完整**，与「好消息/坏消息」表态并列必查。

---

### C2 结尾指向行动

**禁令**：章节结尾禁止抽象总结。

**正向**：最后一句必须是具体的下一步行动或指向下一章的悬念。

**阈值**：章尾末句必须含动词+具体对象。**下一章钩子（P1 强制，非终章）**：末段或末句须含指向下文线索（如「下一章我们将……」「要想弄清XX，请看第 N 章」）；与 §C5 末段钩子一致，**终章豁免**。

| AI味 | 去AI味 |
|------|--------|
| "综上所述，AI将为企业带来巨大变革。" | "现在打开后台，建一个测试机器人。30分钟后你就知道答案。" |
| "本章介绍了三种方法，可根据实际情况选择。" | "三种方法里，先试第二种。原因？门槛最低，当天见效。" |
| 章末戛然而止、无下文预告（非终章） | 「下一章我们拆一下检索账本：你怎么证明这章的每个数字都经得起追问？」 |

---

### C3 打破结构均匀

**禁令**：禁止所有小节结构对称（每节都是"定义→例子→总结"三段论，每节字数相近）。

**正向**：节奏有起伏。有的节200字点到即止，有的节800字深挖，有的节只是一张表。

**阈值**：
- 同章内各节字数标准差 ≥ 均值的30%（即不能全部差不多长）
- 至少1节的结构与其他节明显不同（如其他节是叙述，这节只有一张对比表）

**计算方法**：对每 `###` 节统计汉字数 \(w_i\)，均值 \(\bar w\)，标准差 \(\sigma\)；要求 \(\sigma / \bar w \ge 0.3\)（与 B2-A 变异系数同构，尺度为「节」而非「段」）。

**AI特征（反例：三节对称）**：
```
## 反例·节一 工具A    [300字 定义→例子→总结]
## 反例·节二 工具B    [310字 定义→例子→总结]
## 反例·节三 工具C    [295字 定义→例子→总结]
```

**去AI味（正例：三节不对称）**：
```
## 正例·节一 工具A    [150字 一句话+一张表搞定]
## 正例·节二 工具B    [600字 完整案例深挖]
## 正例·节三 工具C    [80字 "这个别用，原因见正例·节二的教训"]
```

---

### C4 数据具体性

**禁令**：禁止使用整数估计和模糊量词冒充精确数据。

**正向**：真实数据带小数/非整数；无数据时坦诚"没有公开数据"而不是编一个。

**阈值**：整数百分比（如50%/80%/30%等）出现次数 < 全文数据点的20%。**P1 违规（可机读辅助）**：在百分数字段样本 **≥ 5** 的前提下，**整数型「N%」数据 / 全部百分数字段 >20%** 须改写为小数或补来源说明；CLI：`quality-auditor.mjs --int-percent-density`。

| AI味（虚化数据） | 去AI味（具体数据） |
|-----------------|-------------------|
| "效率提升了50%" | "效率提升了47%（基于XX 2025年报告）" |
| "大量企业已经采用" | "截至2026Q1，已有3,247家企业部署（来源：XX）" |
| "节省了80%的时间" | "实测：从4.5小时降到52分钟" |
| 编不出数据 | "目前无公开基准数据，以下为单客户实测" |

---

### C5 章节衔接（v2.1.1 升级：P1 强制）

**目的**：降低「各写各的」导致的断档、重复开篇与无钩子。

> **v2.1.1 变更**：C5 从「多路并行建议项」升级为 **P1 强制**，不区分串行/并行，纳入 C 层计分（共 5 条）。

**检查项（全部必检，任一未达标 = C5 未通过）**：

1. **末段钩子（P1 强制，非终章）**：非终章时，末段或末句须含指向下文线索（如「下一章……」「第 N 章将」「后文……」）；终章豁免。
2. **首段连贯**：开篇须有与上章语境的承接句；引入新概念须先有一句过渡。
3. **跨章重复**：与上章已展开内容复述 ≤ 100 字（超出须改为参见前文）。
4. **卷首过渡（多卷书必检）**：卷首须含过渡导语，说明「上卷讲了什么→下卷要讲什么→为什么关联」。

**阈值**：以上 4 项全达标 = C5 通过；任一未达标 = C5 未通过（❌）。

**与 C2 下一章钩子的关系**：C2 末段钩子要求与 C5 末段钩子检查项重叠——**二者并行检查，C2 未达标时 C5 也视为此项未达标**。

## §B B层·篇级（6条机读项）—— Critic-L3 全文体检


**正向目标：全文读完像一个有经验的人在说话，不像机器在组装**

---

### B0 标题编号唯一性（P0）

**禁令**：全书 `##` / `###` 若采用**多级数字编号**（如 `2.10`、`3.2.1`，须含 **`.`**），同一本书内**不得重复**；否则目录与交叉引用会错位。

**检查**：`node scripts/quality-auditor.mjs --skill-root <智能根> --check-section-ids --glob "**"` 或 `--inputs` 合稿。**逢退出码 1** 表示存在重复键。

---

### B1 标题去公式化

**禁令**：禁止公式化标题模式。

**正向**：标题类型只能是以下三种之一：
- **提问型**：引发读者好奇，如"为什么你的用户流失了？"
- **结论型**：直接给答案，如"用户流失的五大原因"
- **悬念型**：制造好奇，如"第三章的策略为何在第四章被推翻？"

**阈值**：公式化标题 = 0。

| 公式化（AI味） | 去公式化 |
|---------------|---------|
| "XX的重要性" | "为什么XX决定了你能不能活过明天？" |
| "浅谈XX" | 删除"浅谈"，直接说内容 |
| "XX的现状与未来" | "XX：现在能用，但别指望三年后还是这样" |
| "如何利用XX实现XX" | "用XX做XX：三步，不用写代码" |
| "基于XX的XX研究" | 改为人话标题 |
| "一、背景介绍；二、方法论" | 禁止分类标签式标题 |
| "各说考辨""本章总结""阶段一：XXX"——**模板化小标题** | 改为提问/结论/悬念式（例：`杜伯的墓在哪？` 优于 `杜伯研究的主要争议与学术前沿`） |

---

### B2-A 段落节奏

**禁令**：禁止全文段落等长，节奏平坦。

**正向**：段落长度有差异，形成波浪式节奏。

**阈值**：
- 全书段落长度变异系数 ≥ 0.3（避免等长段落）
- 短段落（≤60字）占比建议 15-25%
- 长段落（≥200字）占比建议 10-20%

**检测方法**：
```
变异系数 = 标准差 / 平均值
计算全书所有段落字数的变异系数
```

---

### B2-B 标点多样性

**禁令**：禁止全文只用句号和逗号。

**正向**：标点类型丰富，阅读有节奏感。

**阈值**：每千字使用 ≥ 5种标点类型。

| 标点类型 | 说明 |
|---------|------|
| 句号（。） | 句子结束 |
| 逗号（，） | 停顿 |
| 顿号（、） | 列举 |
| 分号（；） | 分句 |
| 冒号（：） | 引导 |
| 引号（「」或""） | 引用/强调 |
| 感叹号（！） | 情感 |
| 问号（？） | 提问 |
| 省略号（……） | 留白 |
| 破折号（——） | 转折 |

**检测方法**：统计全文标点种类数 / 总字数(千字)

---

### B2-B+ 分省/分地区数据（P1）

> **计分说明**：B2-B+ 是 B2 内的子项，**不独立计入 B 层 6 个机读计分项**（B 层机读计分项为 `B0 / B1 / B2-A / B2-B / B2-C / B3` 共六项；`B2-B+` 作为 `B2-B` 的增强提示，不单独计分）；此项违规作为 P1 质量问题在报告中标注，不单独扣分。


**禁令**：**分省、分市、分地区**等地理切片的**精确数字**（如 `河北 9.4%`）须有**单独来源**（脚注或括注或「数据来源索引」/「本章数据来源索引」中的对应行），**禁止裸数据**。

**正向**：与 [`citation-format.md`](./citation-format.md)、G4 并行默认策略一致；全国总量与分省切片**不得共用一条模糊来源**。

---

### B2-C 结构雷同检测

**禁令**：禁止连续章节/段落使用相同结构。

**正向**：每章有独特开头，每段有不同收尾。

**阈值**：
- 连续3个以上章节使用相同开头结构 → 触发警告
- 连续2个以上段落使用相同结尾句式 → 触发警告

**常见违规模式**：
```
章节开头：
  - "首先……其次……最后……"（三段论式开头连续出现）
  - "让我们来看看……"（万能过渡句连续出现）

段落结尾：
  - "……是非常重要的"（万能判断句连续出现）
  - "……见下文"（引导句连续出现）
```

---

### B3 全局节奏综合

**禁令**：禁止全书节奏单一、语气恒温、风格一致到假。

**正向**：全书有起伏、有变化、有呼吸感。

**阈值**：
- 检测全书章节长度分布曲线，变异系数 > 0.4 时触发警告
- 检测全文语气风格一致性，偏离度 > 0.3 时触发警告
- 全文情绪曲线应有峰有谷，不能全程高能或全程平淡

**检测维度**：

| 维度 | 指标 | 警告阈值 |
|------|------|---------|
| 章节长度 | 各章字数变异系数 | > 0.4 |
| 句式长度 | 平均句长变异系数 | > 0.5 |
| 语气温度 | 情感词密度分布 | 方差过大 |
| 风格一致性 | 用词/句式偏离度 | > 0.3 |

**综合评分（与当前四层等权 10 分制一致）**：
- B层当前脚本口径包含：`B0 / B1 / B2_1 / B2_2 / B2_C / B3`
- B层层分：**通过条数 ÷ 6 × 10**
- 经验通过线：≥ **7.5 / 10**；其中 `B0` 同时也是结构门禁，编号混乱时即使层分不低也应优先修复
- **注**：本节各规则的严重度描述用于解释问题优先级；最终综合分请以 `quality-check.md` §1 与 `metrics.md` 为准。

## §附加 全书级必检项（v2.1.1 新增）

> 以下三项不并入 S/P/C/B 层分，但作为 **S4 质检强制门禁**；任一未通过，S4 结论为**不通过**，不得以综合分掩盖。

### 附加① 承诺兑现率

**检查范围**：`ch00` / `ch01` 中所有「本书将……」「每个 XX 都有……」「你会得到……」等承诺句。

**检查方式**：逐条对照 `.fbs/commitments.md`，核实后续章节是否兑现。

**阈值**：兑现率 = 100%（任何未兑现 = P0，不得判通过）。

**S0 配套**：`commitments.md` 须在 S2→S3 推进前创建，ch00 成文后首批注册，S3 每章成文后更新兑现状态。

---

### 附加② 品牌植入合规

**检查范围**：全书所有工具品牌名（AI 产品名、平台名、SaaS 工具名）。

**阈值**：每章 ≤ 1 次，且仅用于**方法论引用**（「用 X 工具可以……」），不得用于背书或推广语境。

**检查方式**：全文计次，输出超标章节清单。

---

### 附加③ 字数完成度

**检查方式**：当前正文总字数 / `project-config.json` 中 `targetWordCount`。

**阈值**：
- < 50%：S4 **不得判通过**（强制门禁）
- 50%–80%：弱通过，须向用户说明剩余字数缺口
- ≥ 80%：正常进入通过评估
