# 跨章一致性（全书 **CX** · 手册级）

> **定位**：本文件约束 S/P/C/B 各层**单章**为主、**不覆盖章间逻辑**的缺口；**不覆盖** S/P/C/B 二十分；与 **C0** 并列，在**多路并行合稿后、S5 终审前**执行。
> **本文 CX** = 全书跨章层；**勿与** `quality-PLC.md` §**C1**（章内表述层）条款混淆。
> **版本**：3.0.0（CX-4 强制升级 + 分类型章节写作前强制重叠检测门禁）


---

## 1. 何时必须做 CX

| 场景 | 是否执行 CX |
|------|------------|
| 单会话单 Agent 逐章串行，且后章写作时可见前章定稿 | 可选（仍建议 S5 前抽查） |
| **多 Writer 并行写不同章节** | **必须** |
| 对称结构章节（如产品 A vs 产品 B 深度剖析） | **必须**（含对称性视角） |
| **分类型章节（名人/人物/省域/分类型体裁）** | **必须（v2.0.1 升级为 P0）** |

---

## 2. 检查项（CX-1 至 CX-7）

主编以**全书审校角色（Book Auditor）**按下列维度出一份**全书一致性审查报告**（Markdown 即可）：

| 编号 | 内容 | 通过标准（建议） | 优先级 |
|------|------|-----------------|--------|
| **CX-1** | 同一主题在不同章节**论断**是否矛盾（含推测 vs 已证实混用） | 列出矛盾数 = 0 或已标注「估计，待核实」 | P1 |
| **CX-2** | **同一数据**多章出现时数值、单位、口径是否一致 | 不一致数 = 0 或已在附录说明差异 | P1 |
| **CX-3** | **对称章节**小节结构、深度是否大体对等（避免一详一略造成偏斜） | 显著不对称须说明或改稿 | P1 |
| **CX-4** | **重复论述**：核心段落是否在多章完整复述 / **人物/地区在分类型体裁下的跨章重叠** | 后续章改为引用前章或压缩复述；分类型章节见 §CX-4-E | **P0（v2.0.1 升级）** |
| **CX-5** | 前文提出**分析框架**（如对比表、银额清单）后文是否呼应 | 无头无尾 = 0 | P1 |
| **CX-6** | 同一来源在不同章引用**口径**一致 | 冲突须统一 | P1 |
| **CX-7** | **风格一致性**：全书**叙事声调、人称、例证偏好**是否前后统一 | 声调漂移（如前半正式后半口语）须说明或改稿；主动参考 `[S1] 风格档案` 判断 | P1 |

## §CX-4-E 分类型章节人物/地区重叠强制检测（P0，v2.0.1 新增）

> **来源**：2026-04-02 某知识集质检复盘——发现组跨章人物重叠（某月某：Ch36/Ch40；某甲：Ch37/Ch41；某乙：Ch38/Ch42；某预：Ch39/Ch43），且均未在写作前检测，导致同一人物出现两次完整叙述，读者体验割裂，内容价值下降。
> **原有问题**：CX-4 仅 P1 建议级，无写作前强制触发机制，仅在 S5 后期合并时作为可选项检查，已无法阻止重复写入。

### 触发条件（分类型章节写作前自动激活）

```
分类型章节体裁触发词（任一命中）：
- 名人传 / 人物志 / 人物列传 / 分类型人物
- 省域 / 地区 / 城市 / 行政区划
- 分类型体裁（每章一个分类对象，如品牌/公司/产品）
- 章节命名含：{人名}/人物/{地名}/地区/{品牌}/{公司名}
```

### 强制执行流程（P0，写作前门禁）

```
分类型章节写作前（Chapter Brief 输出后、正文成文前）：

Step 1：AI 读取 .fbs/chapter-status.md 中已完成章节清单
         + .fbs/GLOSSARY.md 中已登记的人物/地区名词

Step 2：提取本章 Chapter Brief 中的核心对象（人物名/地区名/分类标签）

Step 3：与台账中的已写清单进行对比：
         - 检查核心对象是否已在任意已完成章节中出现过
         - 检查是否存在地域或人物属性上的高度相似（如同属一省、同一时期）

Step 4（仅在发现重叠时触发）：
  AI 输出重叠预警：
  「⚠️ CX-4-E 重叠预警
    本章核心对象：{本章人物/地区}
    已在 第{N}章 中完整叙述
    建议处理方案：
    A（默认）：本章着重写新角度/补充信息，前章内容以「详见第{N}章」引用
    B：简述概要（≤200字）后跳转前章
    C：本章需要完整叙述（需用户知情确认）

    **福帮手将直接采用方案 A（本章着重写新角度/补充信息，前章内容以「详见第{N}章」引用）**
    如需改用以下方案，请在 Chapter Brief 输出前说明：
    B. 简述概要（≤200字）后跳转前章：「如需详情请参阅第{N}章」
    C. 本章需要完整叙述（用户知情接受重复，写入 project-config.json）
```

**无重叠时**：静默跳过，不向用户输出任何提示，直接进入正文成文阶段。

**有重叠且用户未回复 30 秒**：自动采用方案 A，继续写作。

**方案 C 处理**：将用户知情确认记录写入 `.fbs/project-config.json`：

```json
{
  "cx4e_accepted_overlaps": [
    {"chapter": "Ch40", "object": "某月某", "accepted_by": "user", "timestamp": "..."}
  ]
}
```

### CX-4-E 报告格式（S5 或质量报告中必填）

```markdown
## CX-4-E 重叠检测报告

**触发场景**：{分类型体裁描述}

| 发现 | 涉及章节 | 处理建议 |
|------|---------|---------|
| {人物/地区} 在 Ch{NN} 和 Ch{MM} 均有完整叙述 | Ch{NN}, Ch{MM} | 采用方案A/B/C |

**处理结果汇总**：

| 对象 | 首次章节 | 重叠章节 | 处理方案 |
|--------|---------|---------|---------|
| {人物/地区名} | Ch{NN} | Ch{MM} | 已采用A方案/已采用B方案/用户接受(C) |
```

## CX-4 重复论述 + 人物/地区重叠检测

### 重复论述检测

```
检查范围：核心观点/结论是否在多章完整复述（而非引用/概述）

判断标准：
- 超过 150 字的段落与其他章节高度重叠 → P0 问题
- 核心论断在多章出现但表述不同 → 先确认是否矛盾（CX-1），再压缩为引用

处理方式：
- 保留首次出现的完整表述
- 后续章节改为「如前文第{N}章所述，{核心结论一句话}」
```

## 3. 与 C0、S5 的顺序

```
多路并行合稿完成
      ↓
CX（跨章一致性，本文件）
      ↓
C0（全书破折号/术语总账，见 book-level-consistency.md）
      ↓
S5 终审（quality-check.md §3 + §S5 四层检查单）
      ↓
交付
```

**CX 与 C0 可并行**；两者均须在 S5 终审**之前**完成。

### CX-7 风格一致性检查细则

| 维度 | 检查方法 | 通过标准 |
|------|---------|---------|
| 叙事声调（正式/口语） | 比较前三章与后三章首段 | 无明显漂移，或漂移有说明 |
| 人称（第一/第二/第三） | 全文搜索人称代词分布 | 全书统一，叙事视角不随意切换 |
| 例证偏好（案例型/数据型） | 统计各章案例数 vs 数据引用数 | 比例偏差 < 40%（若大纲无特殊设计） |
| 开头/结尾节奏 | 检查每章的起笔与收尾格式 | 无章节呈无头无尾状（突然开头/突然结束） |

**风格漂移判断依据**：主动参考 `.fbs/project-config.json` 的 `[S1] 风格档案` 字段（由 S1 阶段锁定）。
