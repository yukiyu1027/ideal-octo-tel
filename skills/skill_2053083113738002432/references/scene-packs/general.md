## S0 初始化收集要素

写书开始前，需收集以下信息（标注★为必须）：

1. ★ 书名方向与核心主张（你想通过这本书传递什么？）
2. ★ 目标读者（谁会读这本书？他们面临什么问题？）
3. ★ 体裁与篇幅（书籍/手册/指南，预计多少字/章节）
4. 现有原料情况（是否有素材/访谈/草稿可以盘点）
5. 写作深度档位（轻度/标准/完整，见 SKILL.md §8）
6. 出版意向（自出版 / 内部发行 / 传统出版）

## S2 大纲结构

**通用三层大纲（适用于大多数书籍类型）**

```
前言/序言（可选）
  ├── 成书缘起（1–2段）
  └── 读者使用指南（如何阅读本书）

第一章至第N章（正文核心）
  ├── 章节开头：引言（背景铺垫 + 本章结构预告）
  ├── 各节：论点 → 论据/案例 → 分析 → 小结/过渡
  └── 章节结尾：要点回顾（2–5条）+ 下章预告

后记/结语（可选）
  ├── 全书要点总结
  └── 对读者的期望/下一步行动建议

附录（按需要）
  ├── 术语表
  ├── 参考文献录
  └── 工具/模板
```

## S/P/C/B 质量规则

### S 层（句级）

- **S1** **[MUST]** 每个核心观点有至少一个具体案例或数据支撑
- **S2** **[MUST]** 语言规范，无明显病句，标点使用符合中文排版规范
- **S3** **[SHOULD]** 开头不用"在当今时代""随着社会发展"等套话开场
- **S4** **[SHOULD]** 不使用"众所周知""显而易见"等无效表述

### P 层（段级）

- **P1** **[MUST]** 段落结构完整：论点句 → 论据/案例 → 分析 → 小结/过渡
- **P2** **[MUST]** 段落长度适中（一般 100–300 字，避免超长堆砌或过短跳跃）
- **P3** **[SHOULD]** 章节间有自然过渡，前后观点不矛盾

### C 层（章级）

- **C1** **[MUST]** 大纲与目录一致，章节编号连续无跳号
- **C2** **[MUST]** 每章正文不低于约定字数（S2 大纲确认时锁定）
- **C3** **[MUST]** 章节开头说明本章核心议题，结尾有要点回顾
- **C4** **[SHOULD]** 关键数据点有图表或表格辅助

### B 层（篇级）

- **B1** **[MUST]** 全书主题高度聚焦，不偏离核心主张（跨章一致性）
- **B2** **[MUST]** 前言承诺与正文内容兑现（说要讲的都讲了）
- **B3** **[MUST]** 全书使用统一术语体系，核心概念首次出现时定义
- **B4** **[SHOULD]** 书名/章节标题与内容高度匹配，无"标题党"嫌疑

## 检索策略

| 阶段 | 检索重点 |
|------|---------|
| S0 | 同类书竞品分析、目标读者画像、市场空白 |
| S1 | 核心论点支撑数据、权威来源核实 |
| S3 | 各章核心事实查证（每章 2–3 次） |
| S5 | 终审核实（关键数据、引用来源） |

**优先来源**：权威机构报告 > 政府/学术机构发布 > 知名媒体深度报道 > 一手原始数据
