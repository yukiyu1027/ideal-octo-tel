# 调查报告场景包——本地内容置入规范（降级用）

> **主体：** report · **细分领域：** 100 细分  
> **说明：** 本文件为四层降级路径第 2 级底层内容。网络不可用或无在线场景包时自动使用。  
> **适用人群：** 调查记者 / 非虚构写作 / 深度内容团队  
> **版本：** 2.0

> **与 whitepaper 的核心区别（FIX-P2-01）**：
> - **report（调查报告）**：以"事实核查"为核心——多方资源交叉核实、当事方回应记录、区分事实/观点。结构以"引言→背景→调查发现→当事方回应→影响分析"为主线。适用：新闻调查、非虚构写作、深度稿。
> - **whitepaper（白皮书）**：以"行业洞察"为核心——现状分析、趋势推演、策略建议，数据引用须3类来源。结构以"现状-趋势-建议"为主线。适用：行业报告、政策分析、市场研究。
> - **核心质量规则差异**：report 强调"多方核实+当事方回应"，whitepaper 强调"量化表达+结论可追溯"。

## S0 初始素材收集要点

1. ▶ 报告主题与核心议题（如："XX行业数据虚构内容调查"）
2. ▶ 调查范围与边界（时间范围 / 地域范围 / 涉及主体）
3. ▶ 已有资源情况（受记者意愿 / 文件资料 / 公开数据）
4. 目标发布平台（传统媒体 / 网络媒体 / 内部报告）
5. 法律敏感性评估（是否涉及法律风险、匿名信源保护要求）
6. 预计篇幅与交付格式

## S2 大纲结构（金字塔 + 调查叙事）

| 节 | 章节 | 字数范围 | 必须包括 |
|----|------|---------|---------|
| 1 | 引言（核心发现） | 800—1500字 | 最重要事实先行 / 核心发现 / 报告范围声明 |
| 2 | 背景与来龙去脉 | 1500—3000字 | 历史脉络 / 关键方关系图谱 / 前期报告梳理 |
| 3 | 调查发现（主体） | 3000—6000字 | 多方资源交叉核实 / 文件证据 / 数据分析 |
| 4 | 当事方回应 | 1000—2000字 | 各方机构完整表态 / 回应记录 / 无回应说明 |
| 5 | 影响与分析 | 1000—1500字 | 事件影响评估 / 程度性根本 / 类比案例 |
| 附 | 数据记录 | 按需 | 原始数据 / 资源说明 / 参考文献表 |

## S/P/C/B 质量规则

### S 层（内容质量）

- **[MUST]** 核心事实须有两个以上独立资源交叉核实，单一资源结论必须明确标注"据单一资源"
- **[MUST]** 涉及具体人物或机构的指控，须记录是否已向当事方要求回应，无回应须说明联系经过
- **[MUST]** 区分事实描述与观点评论，不得将推测以事实方式表述
- **[SHOULD]** 匿名资源应在文中说明匿名原因（风险披露 / 合同限制 / 其他）

### P 层（段落逻辑）

- **[MUST]** 每个核心指控段落循环：事实描述 → 资源引用 → 当事方独立（三段结构）
- **[MUST]** 数字与数据必须标注来源机构、统计时间、样本范围
- **[SHOULD]** 避免连续段落同一资源，多方资源交叉出现增强可信度

### C 层（文字对齐）

- **[MUST]** 人名、机构名、职位名必须核实准确，不得出现错别字或过期职称
- **[MUST]** 时效性标注：区分历史事实（用过去时）与当前状态（标注截止日期）
- **[SHOULD]** 引语格式统一（直接引用用引号 + 归属；间接引用用"据...表示"）

### B 层（全书一致）

- **[MUST]** 各方机构表态完整、公平，不出现明显单方面偏颇
- **[MUST]** 全书时间线自洽，同一事件在不同章节的时间描述不得突兀
- **[SHOULD]** 结尾回应引言中的核心议题，形成事件闭环

## 检索策略

- 每章最多检索 **5 次**
- 优先来源：官方文件 / 法院记录 / 权威媒体报道 / 学术研究 / 当事方公开声明
- 必须引用：至少 2 个可独立核实的信息
- 禁止以单一网络百科作为核心事实依据
