# 企业培训场景包 · 本地内置规范（降级用）

> **体裁**：training · **积分门槛**：500 积分
> **版本**：2.0
> **说明**：本文件为四级降级路径第 2 级兜底内容。
> **适用人群**：企业培训师 / HRD / 知识管理团队

---

## S0 初始化收集要素

1. ★ 培训主题与目标能力（如："新员工销售技能培训"）
2. ★ 目标学员画像（岗位 / 经验水平 / 团队规模）
3. ★ 输出类型（课程教案 / 员工手册 / 知识库文档）
4. 培训时长（半天 / 全天 / 系列课）
5. 是否需要配套练习题和工具模板
6. 企业内部案例是否可用

## S2 大纲结构

| # | 模块 | 字数范围 | 必须包含 |
|----|------|---------|---------|

## S/P/C/B 质量规则

### S 层

- **[MUST]** 每章必须在开头明确列出 2–5 条学习目标（用"学完本章，你将能够……"句式）
- **[MUST]** 案例须来自真实企业场景（可脱敏处理，但情境必须真实可信）
- **[MUST]** 练习题须提供参考答案，选择题须标注正确选项理由
- **[SHOULD]** 全书知识点前后不出现矛盾（跨章一致性，用户洞察 R2）

### P 层

- **[MUST]** 每个知识点段落结构：定义 → 原理 → 例子 → 小结
- **[MUST]** 案例分析须包含：背景 → 问题 → 解决方案 → 结果 → 启示
- **[SHOULD]** 避免"众所周知""显而易见"等表达，确保内容对目标学员真实有用

### C 层

- **[MUST]** 专业术语首次出现时附括号解释（面向非专业学员时）
- **[MUST]** 练习题指令清晰，不产生歧义
- **[SHOULD]** 标题层级清晰（H1章 → H2模块 → H3知识点）

### B 层

- **[MUST]** 全书使用同一套术语体系，不同章节对同一概念称谓一致（用户洞察 R2）
- **[MUST]** 各章学习目标与章末练习题考察内容对应
- **[SHOULD]** 课程难度循序渐进，从基础到应用有合理梯度

## 检索策略

- 每章最多检索 **4 次**
- 优先来源：企业管理案例库 / 行业标准文件 / 权威培训机构教材 / 学术研究
- 必须引用：至少 1 个来自知名企业的真实案例（可脱敏）
