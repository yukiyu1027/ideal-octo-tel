# 个人出书场景包 · 本地内置规范（降级用）

> **体裁**：personal-book · **积分门槛**：500 积分
> **版本**：2.0
> **说明**：本文件为四级降级路径第 2 级兜底内容。
> **适用人群**：职场精英 / 行业专家 / 个人品牌建设者

---

## S0 初始化收集要素

1. ★ 书名方向与核心主张（你想通过这本书传递什么？）
2. ★ 目标读者（谁会买这本书？他们面临什么问题？）
3. ★ 作者独特优势（你的经历/方法论/洞见是什么？）
4. 书籍类型（专业书册 / 方法论 / 行业洞见 / 实践手册）
5. 预计篇幅（10–20章 / 更长）
6. 出版意向（自出版 / 传统出版社 / 内部发行）

## S/P/C/B 质量规则

### S 层

- **[MUST]** 全书必须构建完整、自洽的理论体系，不得是案例的简单堆砌
- **[MUST]** 核心案例须典型且具代表性，覆盖不同行业/规模/情境
- **[MUST]** 作者须有明确的原创性观点（非已有理论的转述），每章至少1个
- **[SHOULD]** 理论与实践交替呈现，避免纯理论章节超过2章连续出现

### P 层

- **[MUST]** 数据引用须标注来源（年份 + 机构），不得使用过期数据（超3年需特别说明）
- **[MUST]** 方法论步骤须可操作、可复现，不能停留在"应该""建议"层面
- **[SHOULD]** 每章结尾提供"关键要点"摘要（3–5条）

### C 层

- **[MUST]** 全书术语体系一致，核心概念首次出现时定义，后续不再变换表述
- **[MUST]** 章节间有明确的承接过渡，读者能感知内容的逻辑递进
- **[SHOULD]** 适度运用小标题、项目符号、图表增加可读性

### B 层

- **[MUST]** 全书主题高度聚焦，不偏离核心主张（跨章一致性审校，用户洞察 R2）
- **[MUST]** 前言承诺与正文内容兑现（说要讲的都讲了）
- **[SHOULD]** 书名/章节标题与内容高度匹配，无"标题党"嫌疑

## 检索策略

- 每章最多检索 **6 次**
- 优先来源：学术文献（知网/Google Scholar）、行业报告 / 经典著作 / 权威媒体深度报道
- 必须引用：理论章节须引用至少 2 篇学术或权威来源
