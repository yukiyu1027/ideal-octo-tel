# 版本管理指引

> 当 MCP 响应中出现版本相关提示时，必须按本文件指引处理。

---

## 版本过旧（服务端强制拦截）

MCP 响应提示**版本过旧被拦截**时，服务端已拦截本次请求，**必须按以下步骤执行**：

1. **告知用户**：立即回复用户"检测到 Skill 版本过旧，需要更新后才能继续使用"
2. **调用工具**：调用 `check_skill_version` 工具获取最新版本号和安装地址
3. **征得确认**：向用户展示安装地址，**等待用户确认后**再执行 Skill 安装流程完成更新
4. **提示重启**：安装完成后，提示用户重新开始新对话/会话以使新版本规则生效

---

## 有新版本可更新（不拦截，用户可选）

MCP 响应提示**有新版本可更新**时，当前请求不受影响，**按以下步骤处理**：

1. **告知用户**：向用户展示当前版本和最新版本号，询问是否现在更新
2. **等待确认**：用户确认更新后，再调用 `check_skill_version` 工具获取安装地址并执行安装
3. **用户拒绝**：若用户选择暂不更新，继续执行当前操作，不再重复提示

---

## 禁止行为

- **禁止自行修改请求头中的 `X-Skill-Version`** 来绕过检查
- **禁止跳过 `check_skill_version` 工具调用**，直接使用响应中的链接（链接可能过期）
- **禁止在用户未确认的情况下自动执行可选更新的安装**
